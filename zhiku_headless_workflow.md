# 智库工作流文档 (zhiku_headless_workflow.md)

## 目标
同时向5个国产AI平台提问并收集回复，每个步骤和回复独立显示在OpenClaw Web端

## 核心约束
1. **必须使用无头模式** - 通过Browser Relay的CDP连接
2. **必须选择高级模型** - 豆包选"专家"，Kimi选"思考"等
3. **必须准确定位输入框** - 每个平台使用正确的输入方式
4. **不能重复打开页面** - 必须复用已有标签页
5. **不能重复提问** - 创建新对话以避免获取旧缓存回复
6. **最多2个subagent** - 避免API限流

## 平台配置
| 平台 | URL | 高级模式 |
|------|-----|----------|
| 千问 | https://chat.qwen.ai/ | 思考模型 |
| 智谱 | https://chatglm.cn/ | 深度思考 |
| Kimi | https://www.kimi.com/ | Kimi 思考 |
| 豆包 | https://www.doubao.com/chat/ | 专家模式 |

## 2个Subagent分工

### subagent1
- 负责向千问、智谱、Kimi 提问
- 负责收集豆包、DeepSeek的回复
- 输出【豆包】【DeepSeek】【总结】块

### subagent2
- 负责向豆包、DeepSeek 提问
- 负责收集千问、智谱、Kimi的回复
- 输出【千问】【智谱】【Kimi】块

**关键**：两者同时启动，复用同一CDP连接和已有页面

## 执行流程

### subagent1 流程：
```
1. [步骤] 连接浏览器 (CDP)
2. [步骤] 检查/打开千问页面 → 提问 → 已向千问提问
3. [步骤] 检查/打开智谱页面 → 提问 → 已向智谱提问  
4. [步骤] 检查/打开Kimi页面 → 提问 → 已向Kimi提问
5. [步骤] 开始收集回复
6. [步骤] 切换到豆包页面 → 检查是否有新回复 → 【豆包】回复内容 → 报告主agent【豆包】回复内容收集完成并输出内容块到openclaw web端
7. [步骤] 切换到DeepSeek页面 → 检查是否有新回复 → 【DeepSeek】回复内容 → 报告主agent【DeepSeek】回复内容收集完成并输出内容块到openclaw web端
8. [步骤] 如果收集不完整，多轮轮询等待，等待间隔10秒
9. 【总结】收到X/5个平台的回复
10.  向主agent报告所有任务已完成subagent1和2都已退出
```

### subagent2 流程：
```
1. [步骤] 连接浏览器 (CDP)
2. [步骤] 检查/打开豆包页面 → 提问 → 已向豆包提问
3. [步骤] 检查/打开DeepSeek页面 → 提问 → 已向DeepSeek提问
4. [步骤] 开始收集回复
5. [步骤] 切换到千问页面 → 检查是否有新回复 → 【千问】回复内容 → 报告主agent【千问】回复内容收集完成并输出内容块到openclaw web端
6. [步骤] 切换到智谱页面 → 检查是否有新回复 → 【智谱】回复内容 → 报告主agent【智谱】回复内容收集完成并输出内容块到openclaw web端
7. [步骤] 切换到Kimi页面 → 检查是否有新回复 → 【Kimi】回复内容 → 报告主agent【Kimi】回复内容收集完成并输出内容块到openclaw web端
8. [步骤] 如果收集不完整，多轮轮询等待，等待间隔10秒
9. 向主agent报告所有任务已完成subagent2退出
```

## 关键技术点

### 1. 复用已有页面
```javascript
// 查找已打开的页面，而不是新建
async function findOrOpen(url) {
    for (const p of ctx.pages()) {
        if (p.url().includes(url域名)) return p;  // 复用
    }
    // 没找到才新建
    return await ctx.newPage();
}
```

### 2. 检测新回复
- 检查页面内容是否包含问题关键词
- 或检查是否包含"正在搜索""已阅读"等状态
- 避免获取旧缓存

### 3. 选择高级模式
- 每个平台有不同的选择方式
- 需要先点击模型选择器，再选择高级模式

### 4. 准确定位输入框
- 千问：textarea
- 智谱：div[contenteditable]
- Kimi：textarea
- 豆包：textarea
- DeepSeek：textarea
