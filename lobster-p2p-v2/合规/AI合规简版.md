## 一、HITL 三种模式

| 模式 | 自动程度 | 适用场景 |
|------|----------|----------|
| **Full Auto** | 100% AI | 金额 < ¥100 |
| **Approval** | AI 处理，human 确认 | ¥100-1000 |
| **Supervision** | AI 执行，human 监督 | > ¥1000 |

### 触发人工介入的条件

```python
if any([
    transaction_amount > 1000,
    new_service_provider,
    dispute_raised,
    unusual_pattern,
    first_time_user,
]):
    await human_approval()
```

## 二、中国法规合规

### 1. AI 生成内容

| 法规 | 要求 |
|------|------|
| 《生成式AI管理办法》 | 内容需合规，禁止虚假信息 |
| 《互联网信息服务深度合成管理规定》 | 需标识 AI 生成 |

**合规措施**：
- 所有 AI Agent 需明确披露身份
- "我是 AI 助手，替主人处理此事"

### 2. 资金托管

| 法规 | 要求 |
|------|------|
| 《支付牌照》 | 需持牌机构 |
| 《非银行支付机构条例》 | 资金需托管在银行 |

**合规措施**：
- 不自建资金池
- 用银行/持牌机构托管（如支付宝商户版）

### 3. 数据隐私

| 法规 | 要求 |
|------|------|
| 《个人信息保护法》(PIPL) | 需用户授权 |
| 《数据安全法》 | 数据分级保护 |

**合规措施**：
- 用户授权后才能使用数据
- 数据不出境（海外用户数据存当地）

### 4. 灵活用工

| 法规 | 要求 |
|------|------|
| 《劳动法》 | 不构成劳动关系 |
| 《平台经济》 | 合法的自由职业交易 |

**合规措施**：
- 明确是"信息服务"而非"劳务派遣"
- 不对服务商进行人身管理

## 四、技术实现

### AI Agent 架构

```
需求方 AI Agent          服务商 AI Agent
     ↓                         ↓
需求分析 + 审核    ←→    报价 + 交付
     ↓                         ↓
     ←───── Human Loop ─────→
            ↓
       人工确认
            ↓
       资金释放
```

### 审计日志

所有 AI 决策需记录：
- 时间、输入、输出
- 是否触发 HITL
- Human 的决策
