# 豆包提问模板

> 让豆包回答更精准的实用模板集合

## 核心原则

**好的问题 = 好的答案**

| 模糊问题 | 精准问题 |
|---------|---------|
| "Python 怎么学？" | "我每天有2小时，想3个月内学会Python做数据分析，请给我学习路线" |
| "AI 发展趋势" | "我需要了解2026年AI在中国的发展趋势，包括政策、市场、技术三个维度" |
| "帮我写个代码" | "用Python写一个读取CSV文件并生成折线图的函数，数据量约10万行" |

---

## 提问公式

```
[背景] + [需求] + [范围] + [角度] + [格式]
```

### 各部分说明

- **背景**：你是谁/为什么需要这个问题
- **需求**：具体要什么
- **范围**：时间、地点、领域限制
- **角度**：从哪个方面分析
- **格式**：希望用什么形式呈现（表格、列表、代码等）

## 场景模板

### 1. 调研类问题

**适用场景**：了解行业、市场、技术趋势

**模板**：
```
我正在研究 [主题]，请帮我收集以下信息：

1. 当前市场现状和关键数据
2. 最近6个月的重大变化/进展
3. 主要玩家和竞争格局
4. 存在的挑战和机会
5. 未来发展趋势预测

请：
- 优先使用2025-2026年的最新数据
- 用表格汇总关键数字
- 标注信息来源
```

**示例**：
```
我正在研究中国AI Agent市场的发展现状，请帮我收集以下信息：

1. 当前市场规模和增长率
2. 最近6个月的重大产品/融资消息
3. 主要玩家及其核心产品
4. 技术成熟度和落地难点
5. 2026年发展趋势预测

请用表格汇总关键数据，并标注信息来源。
```

### 2. 技术问题

**适用场景**：编程、调试、技术选型

**模板**：
```
我遇到一个 [技术问题]：

【环境】
- 操作系统：[如 macOS]
- 编程语言/版本：[如 Python 3.11]
- 使用的框架：[如 FastAPI]

【问题描述】
- 目标：我想实现 [什么功能]
- 实际结果：[出现了什么错误/问题]
- 错误信息：[如有报错信息]

【已尝试】
- 方法1：[做了什么]
- 方法2：[做了什么]

请：
1. 解释问题原因
2. 提供具体的修复代码
3. 解释代码原理
```

**示例**：
```
我遇到一个 Python 异步编程问题：

【环境】
- 操作系统：macOS
- 编程语言：Python 3.11
- 使用框架：asyncio

【问题描述】
- 目标：并发执行10个API请求
- 实际结果：请求是串行执行的，没有并发效果
- 错误信息：无

【已尝试】
- 方法1：使用 asyncio.sleep 测试，发现是顺序执行
- 方法2：查阅文档但没找到原因

请：
1. 解释为什么没有并发
2. 提供正确的实现代码
3. 解释 asyncio.gather 的工作原理
```

### 3. 学习咨询

**适用场景**：学习路线、技能提升

**模板**：
```
我想学习 [技能/领域]，目标是 [具体目标]：

【当前基础】
- 已掌握：[已有的知识和技能]
- 了解程度：[了解/熟悉/精通]

【学习条件】
- 每天可用时间：[如 1-2小时]
- 学习方式偏好：[视频/文档/实战]
- 是否有基础：[是/否]

【目标】
- 短期目标：[1-3个月达到什么]
- 长期目标：[半年-一年达到什么]

请给我：
1. 学习路线图（分阶段）
2. 每个阶段的核心知识点
3. 推荐的学习资源（免费优先）
4. 实践项目建议
```

**示例**：
```
我想学习大模型应用开发，目标是半年内能够独立开发AI应用：

【当前基础】
- 已掌握：Python 基础，会用 FastAPI
- 了解程度：了解 AI 基本概念，用过 ChatGPT

【学习条件】
- 每天可用时间：2小时
- 学习方式偏好：实战为主
- 是否有基础：有编程基础

【目标】
- 短期目标：3个月内能调用 API 开发简单应用
- 长期目标：6个月能独立完成 RAG、AI Agent 等项目

请给我：
1. 学习路线图（分阶段）
2. 每个阶段的核心知识点
3. 推荐的学习资源
4. 实战项目建议
```

```

【市场背景】

请：
1. 用表格对比主要玩家
2. 分析 [公司] 的差异化策略
3. 给出机会和风险评估
```

## 进阶技巧

### 1. 追问法

先用简单问题启动，再逐步深入：
```
1. "什么是 RAG？"
2. "RAG 有哪些主流实现方式？"
3. "用 Python 怎么实现一个基本的 RAG 系统？"
```

### 2. 分步法

复杂问题拆解：
```
不要问："帮我写一个完整的 AI 聊天机器人"
要问：
- "Python 如何调用 OpenAI API？"
- "如何实现流式输出？"
- "如何添加上下文记忆？"
```

### 3. 约束法

加限制让回答更精准：
```
"用 100 字以内解释..."
"只列出 3 个关键点..."
"用比喻的方式解释..."
```

---

## 注意事项

❌ 不要：
- 问太宽泛的问题（"告诉我关于 Python 的一切"）
- 不提供任何背景
- 同时问多个不相关的问题
- 要求太笼统（"写得好一点"）

✅ 应该：
- 问题具体明确
- 提供必要的上下文
- 一个问题一个问题问
- 说明具体的标准和期望
