# 小龙虾 P2P 聊天

**版本**: 1.0.0  
**描述**: 点对点直接聊天，不依赖任何服务器

## 功能

- 🦞 **P2P 直连**：两个小龙虾之间直接 TCP 连接发消息
- 📡 **Ping 通就发**：先尝试连接对方，连上就发消息，断开
- 👤 **自定义昵称**：设置你的显示名称
- 🏷️ **标签系统**：给自己打标签（如"安装工"、"业主"）
- 📇 **联系人管理**：自动保存已连接过的小龙虾

## 工作原理

```
A 想给 B 发消息
    ↓
A 的小龙虾尝试连接 B 的 IP:端口
    ↓
[成功] → 握手 → 发消息 → 断开
[失败] → 提示连接失败
```

## 首次使用

### 1. 启动 P2P 服务器

首次使用需要先启动 P2P 服务器：
```
启动 P2P
```

### 2. 告诉对方你的地址

查看自己的信息：
```
我的 P2P 信息
```

这会显示你的 IP 地址和端口，**通过微信/飞书发给对方**。

### 3. 连接对方

对方告诉你他的 IP:端口 后，执行：
```
连接 对方IP:端口 消息
```

例如：`连接 192.168.1.100:5566 你好！`

## 命令列表

| 命令 | 说明 |
|------|------|
| `启动 P2P` | 启动 P2P 服务器（首次使用需执行） |
| `停止 P2P` | 停止 P2P 服务器 |
| `我的 P2P 信息` | 查看自己的 IP、端口、设备ID |
| `连接 IP:端口 消息` | 连接对方并发送消息 |
| `我的昵称 名字` | 设置昵称 |
| `我的标签 标签1,标签2` | 设置标签（逗号分隔） |
| `联系人列表` | 查看已保存的联系人 |

## 使用场景

### 安装工找业主
1. 业主告诉安装工：`我的 P2P 信息`
2. 安装工：`连接 192.168.x.x:5566 你好！`
3. 业主收到消息，直接回复

### 业主群（无服务器）
- 大家互相交换 P2P 信息
- 之后直接点对点联系，不经过任何服务器

## 注意事项

- ⚠️ 双方都需要在**同一网络**（局域网）或**都有公网 IP**
- 📝 首次连接需要手动交换地址（微信发一下）
- 🔄 IP 可能会变，变了需要重新交换地址

## 技术细节

- 端口：5566
- 协议：TCP + JSON
- 消息格式：
```json
{
  "type": "chat",
  "from": "设备ID",
  "nickname": "昵称",
  "content": "消息内容"
}
```
