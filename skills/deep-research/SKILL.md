# 深度调研工作流 (Deep Research Skill)

## 适用场景

当用户提出需要调研某个主题时使用。

## 两种调研模式

### 模式一：用户一起参与（协作模式）

**特征**：用户在消息中表现出想一起参与调研（如"帮我查一下 XXX"）

**工作流**：
1. **初步搜索**：派 2-3 个 subagent 并行搜索不同角度
2. **用户反馈**：用户根据初步结果给方向（如"这个方向好，继续深入"）
3. **细化调研**：根据用户方向再派 subagent 深入
4. **方案落地**：整合所有信息，形成具体方案
5. **文档沉淀**：更新到飞书文档 + 本地文档

**关键**：
- 每轮调研后等待用户反馈
- 用户给方向，我执行
- 用户可能补充他们的专业知识

---

### 模式二：用户不参与（自主模式）

**特征**：用户只是说"帮我调研一下 XXX"，没有参与后续

**工作流**：
1. **多角度搜索**：派 3+ 个 subagent 从不同角度搜索
2. **举一反三**：
   - 根据已有信息主动思考可能的盲区
   - 派 subagent 搜索"更优解"、"最新方案"、"替代方案"
   - 使用浏览器搜索最新内容
3. **迭代深入**：
   - 第一轮搜索结果 → 发现新关键词 → 第二轮搜索
   - 不断循环直到信息饱和
4. **总结汇报**：
   - 整合所有发现
   - 给出建议方案
   - 说明可能的盲区待验证
5. **文档沉淀**：更新到飞书 + 本地

**关键**：
- 不要只搜一次就停下来
- 主动寻找更优解和最新信息
- 不断使用 subagent + 浏览器 + web_search 工具
- 参考秘塔 AI 搜索的工作流：自动深入、多轮搜索、整理总结

---

## 调研执行步骤

### Step 1: 理解需求

明确调研目标：
- 核心问题是什么？
- 用户的使用场景？
- 有哪些已知限制？

### Step 2: 第一轮搜索

派 subagent 并行搜索：
- 基础概念和原理
- 主流方案
- 优缺点对比

```python
# 典型 subagent 任务
"Search for: [主题] basic concepts, mainstream solutions, pros and cons"
```

### Step 3: 分析与扩展

- 归类已有信息
- 识别知识盲区
- 列出需要进一步搜索的点

### Step 4: 深入搜索（多轮）

针对盲区继续搜索：
- 最新解决方案
- 替代方案
- 用户可能没提到的需求

### Step 5: 整合方案

- 对比各方案
- 给出推荐方案
- 说明适用场景和限制

### Step 6: 文档沉淀

更新到两个地方：
1. **飞书云文档** - 方便用户查看
2. **本地文档** - `~/.openclaw/workspace/docs/` 目录

## 工具使用建议

调研必须同时调用多种工具，并行执行，不只靠一个平台：

| 工具 | 用途 |
|------|------|
| `zhiku-ask.js` | 五平台并行搜索（DeepSeek/Kimi/Doubao/GLM/Qwen） |
| `tavily-search` | 实时网络搜索，获取最新信息 |
| `web_fetch` | 获取官方页面详细内容 |
| `sessions_spawn` (subagent) | 并行搜索不同角度 |
| `web_search` | 快速获取最新信息 |
| `browser` | 用 Chrome 搜索最新内容、处理复杂页面 |

**并行调用原则（必须遵守）：**
当用户说"调研"时，同时启动：
1. `node zhiku-ask.js "[问题]"` — 五平台并行思考
2. `tavily_search` — 实时搜索最新结果
3. `web_fetch` — 获取官方文档
4. 派 3-5 个 subagent 从不同角度搜索

总耗时 = 最慢工具的时间（约 30-60 秒），比串行快 5 倍。

## 秘塔 AI 搜索工作流参考

1. **智能理解**：理解用户真实需求
2. **全网搜索**：多来源、多角度（zhiku五平台 + tavily + subagent）
3. **自动整理**：结构化输出
4. **深度挖掘**：追问扩展
5. **原文引用**：标注来源

**多工具并行调研示例：**
```
用户：调研一下 RTX 5050
→ 我立即并行启动：
  1. zhiku-ask.js "RTX 5050 规格参数、发布时间、价格"
  2. tavily_search "RTX 5050 release date specs price 2026"
  3. web_fetch NVIDIA官方页面
  4. subagent×3 分别搜索：评测/游戏性能、vs AMD竞品、发售日期
→ 汇总所有结果 → 形成完整报告
```

## 注意事项

1. **不要只搜一次就停止** - 主动深入
2. **用户补充信息很重要** - 他们可能有你不知道的专业知识
3. **及时文档沉淀** - 方便后续查找
4. **用中文总结结果** - 确保用户能阅读
5. **说明盲区** - 告诉用户哪些还没验证
