## 方案B：AMD Ryzen AI Halo 128GB（目标方案）

### 硬件规格
- **芯片**: Ryzen AI Max+ 395（Strix Halo）
- **CPU**: 16 核 / 32 线程 Zen 5
- **GPU**: Radeon 8060S，40 CUs RDNA 3.5，60 TFLOPS
- **NPU**: XDNA 2，50 TOPS
- **总 AI 算力**: 126 TOPS
- **统一内存**: 128GB LPDDR5X（256-bit，8533 MT/s）
- **内存带宽**: ~270 GB/s
- **系统**: Windows + Linux 双系统
- **预计售价**: 1500-2500 美元（约 10000-18000 元）
- **上市**: 2026 Q2

### AI 推理能力（关键优势）
**128GB 统一内存 = 可以跑 200B 参数模型**

| 模型 | 能否运行 | 说明 |
|------|---------|------|
| Qwen2.5-7B | ✅ | 轻松 |
| Qwen2.5-14B | ✅ | 流畅 |
| Qwen2.5-32B | ✅ | 流畅 |
| Qwen2.5-72B | ✅ | 可以（占用 96GB）|
| DeepSeek-R1-671B | ✅ 可跑 | 量化后可跑，INT4 约需 192GB |
| GPT-4 级别（无开源） | - | 仍需更大内存 |

### 统一内存架构优势
传统 PC 的内存和显存是分开的（CPU 内存 + GPU 显存），数据需要在两者之间拷贝，形成瓶颈。统一内存则让 CPU 和 GPU 共享同一块内存：

- **优点**：128GB 全都可作为显存使用，无需拷贝
- **对 0-1 的意义**：可以跑真正大的本地模型，延迟更低

### AI Halo 的 ROCm 生态注意事项
- AI Halo 支持 ROCm + Windows + Linux
- LLM 推理：vLLM（ROCm build）+ AMD Quark 量化
- 框架：PyTorch ROCm wheels
- **优势**：128GB 统一内存，跑大模型无显存边界问题
- **劣势**：ROCm 生态不如 CUDA 成熟，部分框架可能有 bug
