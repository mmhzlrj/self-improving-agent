## 1. 硬件通信接口

H2C 支持以下通信方式：

| 接口 | 支持情况 | 备注 |
|------|----------|------|
| **Wi-Fi** | ✅ 内置 | 2.4GHz / 5GHz 双频，内置 WiFi 天线 |
| **USB** | ✅ 有 | USB-A 接口位于机器顶部左侧前端 |
| **蓝牙** | ❌ 无 | 非标配 |
| **有线 LAN** | ❌ 无 | H2D/H2S/H2C 均无以太网口 |
| **SD 卡** | ✅ 支持 | 机身内置 8GB EMMC |

**H2C 关键技术参数**（官方规格）：
- 运动控制器：双核 Cortex-M4 + 单核 Cortex-M7
- 内置存储：8GB EMMC
- 控制方式：触摸屏 / 手机 App / PC 软件（Bambu Studio）

> ⚠️ **重要限制**：H2C 没有直接暴露标准串口 G-code 接口。不同于 Marlin/RRF 打印机，H2C 的底层运动控制不对外开放物理串口，所有外部控制必须通过网络接口（MQTT/HTTP）进行。

## 2. G-code 指令支持情况

### 2.1 G-code 风格

H2C 使用 **Marlin 风格**的 G-code（`gcode_flavor: marlin`），由 Bambu Studio 切片生成。

Bambu 官方维护的 G-code 参考：
- GitHub: [rjduran/bambu-gcode-reference](https://github.com/rjduran/bambu-gcode-reference)
- 官方论坛 G-code 列表：[forum.bambulab.com/t/list-of-supported-g-code-commands/25226](https://forum.bambulab.com/t/list-of-supported-g-code-commands/25226)

### 2.2 通过 MQTT API 发送原始 G-code

H2C **不提供**暴露在外可直接连接的串口，但可以通过 MQTT 发送 `gcode_line` 命令来执行原始 G-code：

```json
{
  "print": {
    "sequence_id": "0",
    "command": "gcode_line",
    "param": "M420",       // 可用 \n 分多行
    "user_id": "1234"
  }
}
```

**支持的典型 G-code/M-code**（部分）：

| 类别 | 命令示例 | 说明 |
|------|----------|------|
| 运动 | `G0` / `G1` | 线性移动 |
| 温度 | `M104` / `M109` | 设置喷嘴温度 |
| 热床 | `M140` / `M190` | 设置热床温度 |
| 风扇 | `M106` / `M107` | 冷却风扇控制 |
| 回零 | `G28` | 原点复归 |
| 打印状态 | `M420` | 查询 BBL 级别信息 |
| 速度 | `M220` | 进料速度调整 |
| 归位 | `M211` | 软件限位开关 |
| Bambu 特有 | `M900 KX.XXXX` | 压力Advance (K值校准) |

> ⚠️ 注意： Bambu 的 G-code 解析不完全开放。部分 Marlin 标准指令可能被 Bambu Studio 的 header/config 块拦截或覆盖。第三方直接发送非标准 G-code 时需注意上下文。

## 3. 开放 API 与控制协议

### 3.1 MQTT API（主要控制接口）

**这是 H2C 控制的唯一公开协议**。Bambu Lab 所有打印机都通过 MQTT 与官方服务器/客户端通信。

#### 连接方式

**方式 A：通过云端（需要互联网）**
```
Broker:   mqtt://us.mqtt.bambulab.com:8883
TLS:      是
用户名:   u_{USER_ID}（通过 API 获取）
密码:     {ACCESS_TOKEN}
```

**方式 B：通过本地 LAN（推荐，延迟低）**
```
Broker:   mqtt://{打印机IP}:8883
TLS:      是
用户名:   bblp
密码:     {LAN_access_code}（在打印机屏幕上获取）
```

> 获取 LAN Access Code：打印机屏幕 → 设置 → LAN Only → 查看访问码

#### 关键 MQTT 命令

| 命令 | 说明 |
|------|------|
| `get_version` | 获取固件版本 |
| `pushall` | 获取完整状态报告 |
| `gcode_line` | 发送原始 G-code |
| `gcode_file` | 执行打印机本地 G-code 文件 |
| `project_file` | 打印 3MF/G-code 项目 |
| `start` / `pause` / `resume` / `stop` | 打印控制 |
| `print_speed` | 设置打印速度 (1-4档) |
| `ledctrl` | 控制 LED |
| `ams_change_filament` | AMS 换料 |
| `calibration` | 校准流程 |
| `unload_filament` | 卸料 |

#### 主题（Topics）

| 方向 | Topic |
|------|-------|
| 发送命令 | `device/{serial}/request` |
| 接收响应 | `device/{serial}/response` |
| 状态推送 | `device/{serial}/report`（需发送 `pushall` 开启）|

> 📖 完整协议文档：[Doridian/OpenBambuAPI](https://github.com/Doridian/OpenBambuAPI/blob/main/mqtt.md)

---

### 3.2 LAN Only + Developer Mode

H2C 必须开启 **LAN Only 模式**才能进行本地 MQTT 连接：

1. 打印机屏幕 → 设置 → LAN Only → 开启
2. 设置 → Developer Mode → 开启（解锁更完整的本地 API）
3. 获取 LAN Access Code

> ⚠️ **限制**：开启 Developer Mode 后，打印机将**断开 Bambu Cloud**，只能工作在 LAN 模式。这是 H2C 本地控制的必要条件。

---

### 3.3 Bambu Connect（第三方集成方案）

Bambu Lab 官方提供 **Bambu Connect**（Beta）作为第三方集成中间件：

- URL Scheme：`bambu-connect://import-file?path=...`
- 第三方软件可通过此协议触发 Bambu Connect 打开文件并发送到打印机
- 需要 Bambu Connect 客户端运行在同网络中

> 📖 文档：[wiki.bambulab.com/en/software/bambu-connect](https://wiki.bambulab.com/en/software/bambu-connect)

### 4.3 自建 MQTT 客户端

由于 MQTT 协议已完全公开，可直接用 `paho-mqtt` 等库自建客户端：

```python
import paho.mqtt.client as mqtt
import json

def on_connect(client, userdata, flags, rc):
    # 订阅状态推送
    client.subscribe(f"device/{serial}/report")
    client.subscribe(f"device/{serial}/response")

def on_message(client, userdata, msg):
    data = json.loads(msg.payload)
    print(json.dumps(data, indent=2))

client = mqtt.Client()
client.username_pw_set("bblp", "your_lan_access_code")
client.tls_set()  # 必须 TLS
client.connect("192.168.x.x", 8883)
client.loop_forever()
```

## 5. 与 Cyber Bricks / ESP32 的集成方案

### 5.1 集成架构分析

**Cyber Bricks V7RC** 是一个轻量级开源实时控制框架，支持 LEGO 风格的模块化控制。将其与 H2C 集成，有以下几种路径：

### 方案 A：ESP32 作为 MQTT 网关（推荐）

```
[Cyber Bricks / ESP32]  -->  WiFi  -->  [H2C MQTT API]  -->  [打印机]
```

- ESP32 连接 WiFi，通过 `paho-mqtt` 或原生 MQTT 实现与 H2C 的通信
- H2C 作为 MQTT Broker，ESP32 作为 Client 发送命令
- 可实现：触发打印、查询状态、控制速度等

**优点**：不需要额外的树莓派/Home Assistant，ESP32 独立工作  
**缺点**：需要 Hacker/LAN Access Code，且 H2C 固件变化可能导致协议失效

**关键代码示例**（ESP32 Arduino MQTT 发送 G-code）：

```cpp
#include <WiFi.h>
#include <PubSubClient.h>

const char* ssid = "YOUR_WIFI";
const char* password = "YOUR_WIFI_PASSWORD";
const char* mqtt_server = "192.168.x.x";    // H2C IP
const char* mqtt_user = "bblp";
const char* mqtt_pass = "YOUR_ACCESS_CODE";
const char* printer_serial = "YOUR_SERIAL";

WiFiClient espClient;
PubSubClient client(espClient);

void sendGcode(const char* gcode) {
  char topic[128];
  char payload[512];
  sprintf(topic, "device/%s/request", printer_serial);
  sprintf(payload,
    "{\"print\":{\"sequence_id\":\"0\",\"command\":\"gcode_line\","
    "\"param\":\"%s\"}}", gcode);
  client.publish(topic, payload);
}

void setup() {
  // ... WiFi & MQTT connect ...
  sendGcode("M105"); // 查询温度
}
```

### 方案 B：Cyber Bricks → Home Assistant → H2C

```
[Cyber Bricks]  -->  ESPHome  -->  [Home Assistant]  -->  [MQTT]  -->  [H2C]
```

- Cyber Bricks V7RC 支持 ESPHome，可接入 Home Assistant
- Home Assistant 通过 [ha-bambulab](https://github.com/greghesp/ha-bambulab) 集成控制 H2C
- Cyber Bricks 可通过 Home Assistant 自动化间接控制打印机

> 🔗 Cyber Bricks + ESPHome 讨论：[forum.bambulab.com/t/interfacing-cyberbricks-with-esphome/172218](https://forum.bambulab.com/t/interfacing-cyberbricks-with-esphome/172218)

### 方案 C：USB 串口（受限制）

H2C 的 USB-A 接口主要用于：
1. **U盘打印**：将 G-code 文件拷贝到 U 盘，插入 USB 接口
2. **理论上支持 ADB/调试**（可能需要特殊固件）

> ⚠️ H2C USB 接口**不暴露标准串口**，不能像 Marlin 打印机那样用 USB 直连 ESP32 发送 G-code。

### 5.2 Cyber Bricks 集成路径总结

| 路径 | 复杂度 | 延迟 | 可靠性 | 推荐场景 |
|------|--------|------|--------|----------|
| ESP32 MQTT 直连 H2C | 高 | 低 | 中（协议依赖固件） | 独立项目 |
| Cyber Bricks → ESPHome → HA → MQTT → H2C | 中 | 中 | 高 | 智能家居集成 |
| Cyber Bricks → 树莓派 → Bambu Studio LAN | 低 | 低 | 高 | 原生方案 |

## 6. 技术方案推荐

### 6.1 最小化集成方案（推荐起步）

**ESP32 + MQTT 方案**：

1. 在 H2C 开启 LAN Only + Developer Mode，记录 IP 和 Access Code
2. ESP32 连接 WiFi，用 PubSubClient 连接 `mqtt://H2C_IP:8883`
3. 通过 `gcode_line` 命令发送 G-code，通过 `report` topic 接收状态

```python
# Python 测试脚本（PC 端）
import paho.mqtt as mqtt
import json

broker = "192.168.1.100"  # H2C IP
port = 8883
serial = "XXXXXXXXXX"     # 打印机序列号
code = "ABCDEF"           # LAN Access Code

client = mqtt.Client()
client.username_pw_set("bblp", code)
client.tls_set()  # 必需

def on_connect(client, flags, rc):
    print("Connected")
    # 订阅状态
    client.subscribe(f"device/{serial}/report")
    client.subscribe(f"device/{serial}/response")

def on_message(client, userdata, msg):
    print(f">>> {msg.topic}: {msg.payload.decode()}")

client.on_connect = on_connect
client.on_message = on_message
client.connect(broker, port)
client.loop_forever()
```

## 7. 已知限制与风险

| 风险项 | 说明 | 规避方式 |
|--------|------|----------|
| 固件更新破坏协议 | Bambu Lab 可能随固件更新改变 MQTT 协议 | 锁定固件版本 / 监控协议变化 |
| Access Code 失效 | LAN Access Code 可能过期或重置 | 每次使用前重新获取 |
| Developer Mode 断开云端 | 开启后无法使用 Bambu 官方 App 远程控制 | 根据场景选择：独立运行 vs 联网控制 |
| USB 不暴露串口 | 无法通过 USB 直连 ESP32 发 G-code | 必须用 MQTT WiFi 方案 |
| 无以太网口 | 无法有线直连局域网 | 通过 WiFi 或 USB 共享网络 |
| Bambu Studio 网络插件闭源 | 部分 LAN 功能依赖非开源 DLL | 使用社区开源 MQTT 方案绕过 |

## 9. 结论

**H2C 是高度封闭的生态系统**，没有开放的标准串口 G-code 接口，所有外部控制必须通过网络 MQTT API 进行。

**最适合 Cyber Bricks 集成的路径**：
1. **短期**：ESP32 直连 H2C MQTT — 最直接，但协议稳定性依赖 Bambu 固件
2. **中期**：Cyber Bricks → ESPHome → Home Assistant → MQTT → H2C — 更稳定，适合智能家居场景
3. **长期**：推动 Bambu Lab 开放更多本地 API，或等待社区 Reverse Engineering 的进展

**核心前置条件**：必须开启 **LAN Only + Developer Mode**，获取 **LAN Access Code** 和 **打印机序列号**。
