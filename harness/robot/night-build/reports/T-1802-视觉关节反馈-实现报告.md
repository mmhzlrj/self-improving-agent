## 1. 需求背景

Cyber Bricks 现有控制架构只有 PWM 开环控制，无角度反馈：

```
OpenClaw → MQTT → ESP32-C3 → L298N电机驱动 → 直流电机
                                    ↓
                            只有 PWM 转速控制（无角度反馈）
```

本任务目标：用 ESP32-Cam（OV2640）实现关节角度视觉估算，构建闭环控制，
验证误差 < 5°。

## 3. 实现方案（Q2方案E — 几何视觉标定）

### 3.1 方案选择理由

| 方案 | 精度 | 成本 | 难度 | 适合场景 |
|------|------|------|------|---------|
| 方案A 电流检测 | 堵转检测 | $2 | ★ | 快速验证 |
| 方案B 视觉+ML | ±5° | $0 | ★★★ | Jetson Nano 太慢 |
| **方案E 几何标定** | **±3–5°** | **$0** | **★★** | **本任务采用** |
| 方案D 连杆+深度相机 | ±2–5° | $0 | ★★ | 需要深度相机 |

方案E 核心思路：
- 在每个关节/连杆上贴 ArUco 标记（ID=0~3）
- 标定相机内外参，建立相机→机器人基座坐标系变换
- 从图像中检测 ArUco 角点 → PnP求解 → 相机坐标系3D坐标
- 通过 Homography 将相机坐标转换为基座坐标
- 几何逆运动学（解析公式，无需ML）估算关节角度

### 3.2 标定方法详解

#### 内参标定（calibrate.py --mode intrinsics）
1. 将 ESP32-Cam 安装在固定位置（建议：机械臂正上方，距离 30–50cm）
2. 准备 11×8 角点棋盘格，ESP32-Cam 对准棋盘
3. 采集 20 张不同角度的棋盘格图像（SPACE 键采集）
4. OpenCV `calibrateCamera` 计算内参矩阵、畸变系数
5. 结果缓存到 `intrinsics.npz`

#### 外参标定（calibrate.py --mode extrinsics）
1. 在机械臂每个连杆上贴 ArUco 标记（6×6, ID=0~3）
   - Marker 0: 基座（机器人原点）
   - Marker 1: 肩关节
   - Marker 2: 肘关节
   - Marker 3: 腕关节
2. 将机械臂放置在 HOME 位置（所有关节 90°）
3. ESP32-Cam 拍摄一次，按 ENTER 捕获
4. OpenCV `solvePnP` 估算每个标记的 6-DOF 位姿
5. Umeyama 相似变换对齐，得到 `H_camera_base`（基座→相机）变换矩阵
6. 结果保存到 `calibration_data.json`

## 5. 代码架构

### 5.1 calibrate.py

```
calibrate.py
├── --mode intrinsics  → capture_chessboard_frames() + compute_intrinsics()
│   └── 依赖: OpenCV, numpy, chessboard 标定板
├── --mode extrinsics  → load_intrinsics() + compute_extrinsics_from_markers()
│   ├── ArUco 检测: cv2.aruco.detectMarkers()
│   ├── 单目PnP: cv2.solvePnP() → rvec, tvec
│   └── Umeyama对齐: cv2.estimateAffinePartial3D() → H_camera_base
└── --mode full        → intrinsics + extrinsics → calibration_data.json
```

**关键API（不存在的API检查）**：
- `cv2.aruco.Dictionary_get()` — ✅ OpenCV 官方API
- `cv2.aruco.detectMarkers()` — ✅ OpenCV 官方API
- `cv2.solvePnP()` — ✅ OpenCV 官方API
- `cv2.estimateAffinePartial3D()` — ✅ OpenCV 官方API（3D相似变换）
- `cv2.calibrateCamera()` — ✅ OpenCV 官方API

### 5.2 joint_estimator.py

```
joint_estimator.py
├── RTSPReader (后台线程) → cv2.VideoCapture, 非阻塞读取
├── MarkerDetector       → cv2.aruco + solvePnP → marker_tvecs
├── CyberBricksKinematics
│   ├── estimate_from_3d_points()  → 基于3D坐标的IK（主算法）
│   └── estimate_from_2d()        → 2D fallback（降级方案）
└── AnglePublisher (MQTT)        → paho-mqtt 发布关节角度
```

**几何IK核心算法**（3D模式）：

```
Base Rotation:
  marker_0 在相机坐标系 → 变换到基座坐标系 → atan2(x, y) → base_rot

Shoulder Angle:
  marker_1 在基座Z坐标 → asin((z - L0) / L1) → shoulder

Elbow Angle (余弦定理):
  cos(elbow) = (L1² + L2² - |P2-P1|²) / (2·L1·L2)
  elbow = 180° - acos(cos_elbow)

Wrist Angle:
  forearm_angle - upper_arm_angle → wrist
```

**MQTT主题**：
- 发布: `cyberbricks/joint_states` — JSON: `{"timestamp": float, "joints": {...}}`
- 订阅: `cyberbricks/state`（接收 Cyber Bricks 状态确认）

### 5.3 test_cyberbricks.py

```
test_cyberbricks.py
├── CyberBricksController → MQTT 发布指令 + 等待ACK
├── EstimatorReader       → MQTT 订阅读取估算角度
├── ClosedLoopTest.run()
│   ├── 逐个关节发送目标角度
│   ├── 等待机械臂稳定（settle time 预估）
│   ├── 读取视觉估算角度
│   ├── 计算误差 |commanded - estimated|
│   └── 判定 pass/fail (< 5°)
└── save_reports() → JSON + CSV 详细报告
```

## 6. 使用流程

### 6.1 首次标定

```bash
# Step 1: 内参标定（采集20张棋盘格）
export ESP32_CAM_RTSP="rtsp://192.168.1.100:8554/mjpeg/1"
python3 calibrate.py --mode intrinsics --rtsp $ESP32_CAM_RTSP

# Step 2: 外参标定（机械臂在HOME位置，按ENTER捕获）
python3 calibrate.py --mode extrinsics --rtsp $ESP32_CAM_RTSP

# 标定完成后生成:
#   intrinsics.npz       (内参)
#   calibration_data.json (完整标定数据)
```

### 6.2 启动估算器

```bash
# 基础运行（显示预览窗口）
python3 joint_estimator.py \
  --rtsp rtsp://192.168.1.100:8554/mjpeg/1 \
  --headless  # Jetson Nano 无头模式

# 带MQTT发布（发送到Cyber Bricks MQTT broker）
python3 joint_estimator.py \
  --mqtt --mqtt-host 192.168.1.50 \
  --mqtt-topic cyberbricks/joint_states
```

### 6.3 闭环验证

```bash
# 前提：joint_estimator.py 以 --mqtt 模式运行

# 完整测试（所有关节，3个位置，各重复1次）
python3 test_cyberbricks.py \
  --mqtt-host 192.168.1.50 \
  --joints shoulder elbow wrist \
  --positions 45 90 135

# 单关节测试
python3 test_cyberbricks.py \
  --mqtt-host 192.168.1.50 \
  --joints shoulder \
  --positions 60 90 120

# 输出: JSON + CSV 报告 (night-build/reports/)
```

## 7. 误差分析与优化方向

### 7.1 当前方案误差来源

| 误差来源 | 量级 | 缓解方法 |
|---------|------|---------|
| ArUco 角点检测精度 | ±1–2 px | 增大标记尺寸（40mm+）、亚像素角点 refinement |
| 单目深度估计 | ±5mm @ 30cm | 外参标定要精确；安装距离保持 30–50cm |
| 相机安装位置漂移 | 累积误差 | 机械固定；定期重新标定 |
| 链接长度标定不准 | 关节角度误差 | 用数显游标卡尺精确测量 L0–L3 |
| 光照变化 | 标记检测失败 | 使用磨砂哑光标记；红外照明（可选） |

### 7.2 优化方向（后续任务）

1. **标定优化**：用张正友标定法精确内参；多位置 Umeyama 平均
2. **标记升级**：换用 AprilTag（精度更高）+ 编码标记（消除歧义）
3. **降噪滤波**：对连续帧的角度估计做卡尔曼滤波，减小抖动
4. **深度融合**：结合 RealSense 等深度相机，提升 Z 轴精度
5. **AS5600 编码器**（Q1 推荐）：直接检测电机轴角度，精度 ±0.5°
