# 像素艺术 3D 游戏调研报告
## Stardew Valley / Octopath Traveler 风格

---

## 1. 色彩风格与调色板选择

### Stardew Valley（星露谷物语）
- **高饱和度的柔和色调**：游戏采用温暖、饱和度高的配色，营造田园牧歌感
- **季节性色彩变化**：春夏明亮翠绿，秋冬棕黄暖色，色彩随季节系统自然过渡
- **限制色板策略**：同一场景内严格控制主色调数量，避免颜色混乱
- **昼夜循环独立色板**：不通过代码压暗颜色，而是为夜晚场景单独设计一套色板（避免像素颜色变脏）

### Octopath Traveler（歧路旅人/八方旅人）
- **HD-2D 视觉风格**：将 16-bit 像素角色置于 3D 场景中，形成独特的视觉层次
- **Cinematic Color Palette**：参考电影配色，大量使用金色（阳光）、蓝色（阴影）、紫色（神秘）三大主色
- **光照染色技术**：通过精确的光影计算，让相同像素在不同光照角度下呈现微妙颜色变化
- **Jazz-inspired 配色**：每个城镇根据地理/文化特征有独立配色，如沙漠黄、城郊绿等

### 通用色彩技巧
- **像素色数限制**：常见 16 色、32 色或 64 色调色板
- **避免渐变**：用相邻色块模拟渐变效果
- **色彩心理学**：暖色调=友好/探索，冷色调=危险/战斗

## 2. 3D 世界布局与视角设计

### Stardew Valley 布局特点
- **固定俯视角（Isometric-ish）**：45度角俯视，单一视角简化碰撞检测和寻路
- **农场分区布局**：耕地、畜舍、矿山、沙滩等按功能分区，边界清晰
- **无缝世界**：无需加载屏幕，通过玩家移动触发区域切换
- **Grid-based 摆放系统**：所有建筑/作物按网格对齐，降低设计复杂度

### Octopath Traveler 布局特点
- **动态运镜（Dynamic Camera）**：
  - 移轴摄影（diorama/parallax）效果：分层3D背景+2D像素前景
  - 战斗场景的"闪切"和极短促缩放动画，营造扎实打击感
  - 无多余拖尾动画，拳拳到肉
- **环境叙事**：
  - 树叶随微风摇曳，阳光透过窗户有灰尘粒子效果
  - 云朵飘过在草地上投下实时移动阴影
  - 3D引擎实现纯2D难以做到的环境细节

### 视角设计建议（0-1项目）
| 方案 | 优势 | 劣势 |
|------|------|------|
| 固定视角（Stardew）| 镜头系统简单，无穿模问题 | 视觉表现力有限 |
| 动态运镜（Octopath）| 电影感强，表现力丰富 | 镜头碰撞/透视计算复杂 |

**建议0-1项目选固定视角**，专注于核心玩法和碰撞体搭建

## 3. 动画与交互特点

### Stardew Valley 动画
- **周期循环动画**：角色待机时上下浮动，行走时四肢小幅度摆动
- **帧数风格**：6-8fps 的角色动画，保持像素感
- **交互反馈**：收获/使用道具时有明显的弹出动画和音效

### Octopath Traveler 动画
- **模块化动作表演**：骨骼动画或简单周期性运动，避免逐帧绘制的巨大工作量
- **环境动画**：3D引擎带来的环境细节（树叶、粒子、阴影）
- **视角即玩法**：《卡带小子》中，旋转视角触发物体"存在/消失"状态变化

### 交互设计原则
- **干脆利落**：UI切换0.1秒内，包含透明度变化+弹性缩放
- **反馈即时性**：每个操作都有视觉/音效反馈
- **菜单动画精致感**：参考《歧路旅人2》，菜单切换也有微小动效

## 4. 0-1 项目借鉴建议

### 技术路线：3D模型转像素（推荐）

**为什么？**
- 直接手绘2D像素动画（逐帧）对复杂角色动作（攀爬、游泳、战斗）工作量巨大
- 3D低模+像素着色器可大幅降低美术成本

**做法：**
1. Blender 或 Unity 中建立简单的 3D 低模（Low Poly）
2. 使用特定渲染管线或着色器（Shader）
3. 将 3D 模型渲染成低分辨率像素画面
4. 烘焙成序列帧或直接实时渲染

**参考案例**：《潮痕》开发日志推荐这种方法为"懒省事、好白嫖（复用3D资源）、适合开摆（掩盖细节瑕疵）"

### 开发优先级建议

```
Phase 1: 固定视角 + Grid布局
    ↓
Phase 2: 色彩系统（独立日夜色板）
    ↓
Phase 3: 基础动画（待机浮动 + 行走摆动）
    ↓
Phase 4: 交互反馈（UI动效 + 道具弹出）
```

### 关键设计原则
1. **严格限制主色调数量**（建议 16-32 色）
2. **为每个时间/区域准备独立色板**，不用代码压暗
3. **优先实现干脆的交互反馈**，动画帧数宁可少也不要拖沓
4. **Grid-based 摆放系统**贯穿始终，降低所有决策成本
