# QQ养虾空间 UI 设计调研报告

> 调研时间：2026-03-28
> 调研平台：Exa（via mcporter）
> 注意：QQ养虾空间具体产品信息较少，以下内容综合 QQ 小窝、超级 QQ 秀、腾讯虚拟世界产品线及公开设计资料整理。

---

## 一、产品背景

**「QQ养虾空间」** 是腾讯 QQ 生态下的虚拟社交空间/小游戏功能，与腾讯"鹅虾"（E-gigator/Shrimp）品牌有一定关联。腾讯于 2026 年 3 月 9 日发布了两款 AI + 虚拟空间产品：
- **WorkBuddy**：兼容 OpenClaw 技能，支持 QQ/飞书/钉钉接入的 AI Agent 桌面平台
- **QClaw**：可接入微信的 AI Agent

更早之前，QQ 已有成熟的虚拟世界产品线，包括：
- **QQ 小窝**：用户可装扮的虚拟个人房间
- **超级 QQ 秀**：3D 版 QQ 秀，支持捏脸、换装、空间装扮、小游戏

## 二、关键设计元素

### 1. 色彩体系

| 元素 | 描述 |
|------|------|
| 主色调 | 年轻活力导向，QQ 品牌蓝 + 多巴胺彩色系 |
| QQ 9 主色 | 腾讯家族蓝，渐变过渡到靛蓝/青色 |
| 超级 QQ 秀 | 多色彩、卡通感、饱和度偏高，营造轻松氛围 |
| QQ 阳光牧场 | 清新的阿尔卑斯田园色调，绿色为主 |
| UI 风格 | 年轻科技感、有活力、色彩鲜明 |

### 2. 布局结构

**QQ 小窝/超级 QQ 秀 典型布局：**
```
┌─────────────────────────────────────┐
│           顶部状态栏                  │
├────────┬────────────────────────────┤
│        │                            │
│  侧边  │       3D 虚拟场景主区域      │
│  导航  │    (可交互的虚拟房间/世界)    │
│        │                            │
├────────┴────────────────────────────┤
│           底部功能栏                  │
│  [个人] [商店] [游戏] [社交] [我的]   │
└─────────────────────────────────────┘
```

**WorkBuddy（AI Agent 页）布局：**
- 左侧：新建任务 / Claw / 插件 三个入口
- 主区域：代码开发 / 日常办公 两大功能分区
- 插件面板：内置数十种处理不同任务的插件

### 3. 交互风格

| 交互特点 | 说明 |
|----------|------|
| 虚拟形象（Avatar） | 3D 可定制形象，支持 AI 捏脸（面部识别自动生成） |
| 空间编辑态/运行态 | 可编辑模式（装扮）和运行模式（交互）分离 |
| 物品交互 | 拾取、坐、躺等动作反馈 |
| 社交逛店 | 可看到其他用户虚拟形象，同步社交 |
| 沉浸感 | 虚拟与现实边界模糊，强调"在场感" |

### 4. 功能列表（虚拟世界类）

| 功能模块 | 描述 |
|----------|------|
| 虚拟形象定制 | 3D Avatar，捏脸/换装/配饰 |
| 个人空间（小屋） | 可装扮的虚拟房间，编辑+运行双模式 |
| 逛逛场景 | 官方固定场景，用户可进入互动，无法编辑 |
| 家具商店 | 虚拟物品购买、试用体验（坐沙发、躺床上） |
| 滨海音乐广场 | 大型多人在线空间，类似现实广场 |
| 影音室 | 模板化图片/视频拍摄，在线看视频社交 |
| 小游戏 | 内嵌休闲小游戏（套牛、俄罗斯方块等） |
| AI Agent 能力 | 代码开发、文档总结、数据分析、定时任务 |

### 5. 动画与动效

- **流畅 3D 场景切换**：房间进入/退出有过渡动画
- **物品交互反馈**：拾取、放置有弹性动效
- **虚拟形象动作**：挥手、打招呼等多种预设动作
- **UI 微交互**：卡片悬停、按钮反馈等现代动效
- **情绪色彩**：整体轻松、有趣、年轻化

## 三、技术实现参考（ZEGO 方案）

QQ 小窝类虚拟空间技术架构（参考 ZEGO MetaWorld SDK）：

```
用户端
  ├── 进入场景（资源 ID / 场景配置）
  ├── 运行态：与物品交互（拾取/坐/躺）
  └── 编辑态：添加/移除/移动物品，保存到本地缓存
```

**核心 API：**
- `enterWithHomeAssetId`：通过房间资源 ID 进入场景
- `switchHomeState`：编辑态 ↔ 运行态 切换
- `applyAssetWithAssetId`：添加物品到场景

## 四、对 0-1 项目 3D 世界的借鉴建议

### 1. 色彩与风格
- ✅ 采用**年轻化、活力感**的配色，避免沉闷
- ✅ 善用**渐变和多彩**提升科技感和趣味性
- ✅ 田园/自然主题适合休闲类虚拟世界（参考 QQ 阳光牧场）

### 2. 交互设计
- ✅ **双模式设计**（编辑态/运行态）是必备，降低用户焦虑
- ✅ **Avatar 系统**是虚拟世界身份认同的核心，建议支持 AI 捏脸
- ✅ **物品交互**（拾取、坐、躺）增加沉浸感，动作反馈要流畅

### 3. 社交设计
- ✅ **多人在场感**：能看到其他用户虚拟形象，极大提升社交体验
- ✅ **逛店/逛空间**场景：社交 + 消费自然结合
- ✅ **语音/文字/表情**多维度社交方式

### 4. 功能优先级建议
1. **Avatar 系统**（捏脸+换装）- 身份认同
2. **个人空间**（小窝/房间）- 归属感
3. **物品交互系统** - 沉浸感
4. **社交逛店** - 社交粘性
5. **小游戏/任务** - 可玩性
6. **AI Agent 集成** - 差异化竞争力

### 5. 技术建议
- ✅ 使用成熟 3D 引擎（Unity/Unreal/Three.js）
- ✅ 本地缓存用户编辑状态，减少加载时间
- ✅ 编辑/运行态分离，API 层清晰
- ✅ 移动端做轻量入口，复杂操作在 PC 端
