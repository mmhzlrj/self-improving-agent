> **网页版/像素版资料稀缺**：《模拟人生》网页版（如 Sims Social、Facebook 时代的社交版）及粉丝像素自制项目资料极度零散，缺乏系统性开发文档。以下内容综合《模拟人生》系列正作（Sims 1/2/3/4/ FreePlay）的 UI 设计规律，结合网页/像素版特殊限制进行推断。搜索不到的内容已标注「信息缺失」。

## 一、NPC 行为系统

### 1.1 需求驱动行为（Need-Driven AI）

《模拟人生》系列的核心 AI 逻辑是**需求驱动型**，NPC 自主行为由一套「需求优先级」系统决定。

**典型需求循环（网页版/像素版简化版）：**

| 需求类型 | 图标 | 驱动行为 |
|---------|------|---------|
| 饥饿（Hunger） | 餐具 🍽️ | 自动寻找食物、坐下、吃饭 |
| 精力（Energy） | 灯泡 💡 | 犯困→自动走向床铺→睡觉 |
| 如厕（Bladder） | 马桶 🚽 | 急需时自动中断当前行为去如厕 |
| 社交（Social） | 对话气泡 💬 | 自主找附近 NPC 闲聊 |
| 娱乐（Fun） | 游戏手柄 🎮 | 无聊时自动看电视/玩游戏 |
| 环境（Hygiene） | 浴缸 🛁 | 脏了自动洗澡 |

**网页版/像素版限制处理：**
- 行为树被大幅简化，通常只有 3-5 个核心需求
- NPC 自主行为多为**固定路径巡逻**或**固定点交互**
- 不存在正作的「自由意志」多层决策树

### 1.2 情绪系统（Mood System）

| 特征 | 描述 |
|-----|------|
| 情绪范围 | 极端痛苦（ Miserable）→ 痛苦（ Sad）→ 普通（ Neutral）→ 开心（ Happy）→ 狂喜（ Ecstatic） |
| 颜色映射 | 红（痛苦）→ 橙 → 黄 → 绿 → 蓝（狂喜） |
| 持续时间 | 情绪气泡通常持续显示 3-10 秒后消失 |
| 衰减机制 | 需求不满足时情绪线性下降，满足时快速上升 |

**网页版/像素版特殊处理：**
- 情绪变化通常用头顶气泡颜色直接表达，不显示具体数值条
- NPC 头上出现「!」表示有强烈需求等待满足

## 二、工作动画（Career Actions）

### 2.1 动画类型与表现方式

网页版/像素版受限于资源，工作动画采用**循环动画 + 状态指示**的简化工法：

| 工作类型 | 动画表现 | 状态指示 |
|---------|---------|---------|
| 厨师 | 角色站在炉灶前，手臂上下摆动（翻炒动作） | 食物上方出现飞溅粒子 + 进度条 |
| 医生 | 角色站在病床旁，手臂左右摆动 | 进度条 + 心跳图标 |
| 运动员 | 角色原地跑步姿态（腿部上下跳动） | 汗水粒子特效 |
| 程序员/创作者 | 手臂在胸前敲击键盘姿态 | 代码行滚动（虚构） |
| 服务员 | 托盘举起-放下循环 | 酒杯/盘子图标浮动 |

### 2.2 动画规格（像素版）

| 参数 | 典型值 |
|-----|-------|
| 帧率 | 8-12 FPS（正作 30+ FPS）|
| 循环节拍 | 4-8 帧一个完整动作循环 |
| 悬停特效 | 工作中的 NPC 鼠标悬停显示当前进度 |
| 中断机制 | 点击 NPC 立即中断工作动画，回到 idle 姿态 |

### 2.3 正作参考（FreePlay 版）

《模拟人生：畅玩版》(FreePlay) 对网页版有重要参考价值：

- **社交时间胶囊**：类似状态气泡，NPC 头顶出现想法气泡
- **梦想成真时刻**：重要剧情节点有特殊光效动画
- **工作路线可视化**：地图上显示 NPC 移动路径

## 三、状态气泡系统（Moodlets / Thought Bubbles）

### 3.1 气泡类型分类

| 气泡类型 | 形状 | 内容 | 颜色 |
|---------|------|------|------|
| 需求气泡 | 圆形 + 缺口（对话泡尾巴）| 图标（餐具/床/马桶）| 白底 + 黑边框 |
| 情绪气泡 | 圆形 + 缺口 | 面部表情（笑脸/哭脸/怒脸）| 对应情绪色底 |
| 想法气泡 | 圆形 + 缺口 | 所求物品缩略图 | 白底 |
| 社交气泡 | 椭圆 | 对话文字预览 | 浅蓝底 |
| 特殊气泡 | 菱形/星形 | 重要剧情/buff 提示 | 金色/紫色 |

### 3.2 视觉设计参数

| 参数 | 像素版典型值 | 说明 |
|-----|------------|------|
| 气泡尺寸 | 16×16 / 24×24 / 32×32 px | 根据重要性分级 |
| 边框 | 1-2px 纯色描边 | 黑或深灰 |
| 悬停放大 | 1.5x scale on hover | 提供详细信息 |
| 动画 | 气泡出现：pop-in（弹跳出现）| 持续：轻微上下浮动（2px，1s cycle）| 消失：fade-out（0.3s）|

### 3.3 交互逻辑

```
NPC 空闲 → 头顶冒出需求气泡 → 气泡持续 3-5s
    ↓
玩家点击 NPC → 弹出交互菜单（喂食/聊天/工作/休息）
    ↓
选择交互 → 气泡消失，NPC 执行对应动作
```

## 四、整体 UI 风格

### 4.1 色彩体系

| 色系 | 色调 | 说明 |
|-----|------|------|
| 主色调 | 蓝绿/青蓝 | 经典模拟城市感，科技+生活气息 |
| 强调色 | 橙黄 | 按钮、进度条、通知 |
| 状态色 | 绿→黄→红 | 需求满足度梯度 |
| 背景色 | 浅米/浅灰 | 主界面背景，不抢焦点 |
| 辅助色 | 粉/紫 | 特殊功能、女性化内容 |

**Sims 3 / FreePlay 经典配色：**
- 青色 `#00bcd4`（主操作）
- 橙色 `#ff9800`（强调/警告）
- 绿色 `#4caf50`（正面状态）
- 红色 `#f44336`（负面状态）
- 米白 `#fff8e1`（背景）

### 4.2 布局结构

**经典 Sims UI 布局：**

```
┌─────────────────────────────────────────┐
│  💰金钱    ⏰时间    ☀️/🌙时段     ⚙️设置 │  ← 顶部状态栏
├─────────────────────────────────────────┤
│                                         │
│                                         │
│          3D 虚拟世界主场景               │
│      （等距视角或俯视视角）              │
│                                         │
│   [NPC]  [NPC]         [NPC]           │
│    💬     🍽️            💤              │
│                                         │
├─────────────────────────────────────────┤
│  [角色1]  [角色2]  [角色3]  [+添加]     │  ← 底部角色栏
│   ❤️🧡💛   ❤️🧡💛    ❤️🧡💛            │
└─────────────────────────────────────────┘
```

**网页版特征：**
- 通常采用 **底部角色栏** 设计，所有可控角色排列
- 角色头像下方显示**简化需求条**（3 色条：绿/黄/红）
- 点击角色弹出**圆形径向菜单**（正作经典设计）
- 点击地面触发**移动指令**

### 4.3 圆角与阴影

| 元素 | 圆角 | 阴影 |
|-----|------|------|
| 按钮 | 8-12px | 底部深色投影（模拟按压感）|
| 面板 | 16-20px | 面板叠放时产生层叠阴影 |
| 气泡 | 50%（圆形）| 无阴影，轻微描边 |
| 头像 | 50%（圆形裁切）| 无 |

### 4.4 图标风格

- **像素版**：8-bit / 16-bit 像素图标，简单易懂
- **网页版**：扁平化 SVG 图标，与现代 UI 趋势一致
- **图标库参考**：《模拟人生》系列使用自定义图标集，核心图标约 50-100 个

## 六、对 0-1 项目 3D 世界的借鉴建议

### 6.1 NPC 行为系统

| 建议 | 具体做法 |
|-----|---------|
| **简化需求树** | 控制在 4-6 个核心需求（饥饿/精力/社交/如厕/娱乐/环境），避免过度复杂 |
| **行为优先级队列** | NPC 同时有多个需求时，按优先级排队处理，高优先级中断低优先级 |
| **可视化需求提示** | 用头顶气泡直接表达当前最迫切需求，不需要数值面板 |
| **自主行为 + 玩家指令混合** | NPC 70% 时间自主行为，30% 响应玩家指令 |

### 6.2 工作动画

| 建议 | 具体做法 |
|-----|---------|
| **循环动画 + 粒子特效** | 节省资源，用 4-8 帧循环动画 + 简单粒子表示工作内容 |
| **进度条 + 悬停详情** | 鼠标悬停 NPC 显示当前工作进度条 |
| **夸张有趣 > 真实** | 工作动画可以稍微夸张（出汗、表情变化），增加趣味性 |
| **中断立即响应** | 点击工作中的 NPC 应立即停止动画，切换到 idle |

### 6.3 状态气泡设计

| 建议 | 具体做法 |
|-----|---------|
| **圆形气泡 + 缺口尾巴** | 经典气泡样式，玩家一看就懂 |
| **颜色编码情绪** | 绿=满足/开心，红=不满/急需，注意色盲友好（可加形状区分）|
| **气泡动画轻柔** | pop-in + 轻微浮动，不要突兀 |
| **分层显示** | 多个需求时只显示最高优先级，其他等待中 |
| **想法气泡 > 文字** | 用物品图标代替文字描述，跨语言友好 |

### 6.4 UI 色彩与布局

| 建议 | 具体做法 |
|-----|---------|
| **蓝绿主色调** | 科技+生活气息，经典模拟游戏配色 |
| **底部角色栏** | 触控友好的经典设计，所有角色一览无余 |
| **圆形径向菜单** | 点击角色弹出交互菜单，符合《模拟人生》用户直觉 |
| **顶部状态栏精简** | 只显示金钱、时间、设置，避免信息过载 |
| **扁平化图标** | 统一风格的图标系统，核心 30-50 个 |

### 6.5 圆角与反馈

| 建议 | 具体做法 |
|-----|---------|
| **统一圆角系统** | 定义 4px/8px/16px 三档圆角令牌 |
| **按钮按压感** | 使用底部投影模拟按压感，不需要复杂阴影 |
| **悬停放大气泡** | 悬停 NPC 时放大状态气泡，显示详细信息 |
| **音效辅助** | 不同需求类型配不同音效（饥饿=咕噜声，如厕=冲水声）|
