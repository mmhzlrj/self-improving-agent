## 一、核心设计理念

低多边形（Low-Poly）风格在 3D 城市建造游戏中，已从早期受限于硬件的"技术妥协"演变为一种**主动的美学选择**。

它的核心魅力在于：用**简洁的几何语言**传递直观信息，同时凭借**柔和的色彩**与**物理反馈**，营造出轻松、治愈的沉浸感。

---

## 二、世界布局

### 关键设计原则

| 手法 | 说明 |
|------|------|
| **分区化布局** | 将城市划分为不同功能区（居民区、商业区、地标区），通过主干道连接。主干道在视觉上（宽度、纹理）明显区别于次级小路[citation:8] |
| **地标引导** | 利用高度变化（教堂、雕像置于高地）和标志性建筑（灯塔、钟楼）构建视觉焦点，充当"导航点"引导玩家探索[citation:8] |
| **清晰边界** | 设定物理或视觉边界（海洋、山脉、悬崖）。"有限世界"设计让开发者能精细打磨每一寸土地，给玩家明确的"完成感"[citation:3][4] |
| **程序化生成 + 手工核心区** | 基础地形程序化生成，核心区域手工打造。如《低模之战》随机生成地图，每次体验全新[citation:4] |

### 借鉴建议
对于开放世界项目，建议用**地形（山丘、河流）和道路**自然划分区域，通过**视觉遮挡（树林）**和**地标露出**引导玩家探索未知区域。

## 三、建筑风格

### 核心手法

| 手法 | 说明 |
|------|------|
| **模块化组件** | 提供标准化、可拼接的建筑零件（墙壁、屋顶、楼梯、柱子），玩家像搭乐高一样组合，保证风格统一同时激发创意[citation:9] |
| **主题纯粹** | 聚焦单一主题，如中世纪幻想、威尼斯水城或江南水乡，避免风格杂糅导致视觉混乱[citation:1] |
| **形式追随功能** | 建筑外观直观反映其功能：尖顶+彩色玻璃=教堂；风车+烟囱=工坊[citation:1][9] |
| **无网格自由建造** | 《Townscaper》代表做法：点击即生成，系统根据周围环境自动将方块变成房子、拱门、灯塔或花园[citation:2] |

### 低多边形技术指标（参考）
- 单个模型顶点数：**100-1000 个顶点** [citation:9]
- 技术优势：性能预算极低，节省算力可用于全局光照、动态天气等高级效果

## 四、色彩设计

### 关键手法

| 手法 | 说明 |
|------|------|
| **低饱和度 + 柔和色调** | 主色调使用温暖/清新的柔和色系（米黄、淡蓝、草绿），避免高饱和度色彩带来的视觉疲劳[citation:1][5] |
| **色彩编码** | 用颜色进行功能编码：棕色=木材建筑，灰色=石质结构，绿色=植被区域，帮助玩家快速识别[citation:8] |
| **动态光影** | 昼夜系统（黄昏暖光）与天气系统增强色彩感染力；夜晚窗户透出暖黄光与水面冷色倒影形成对比[citation:1][5] |
| **水彩质感** | 《Water Town》偏向水彩画风格，色彩通透，尤其水的质感（波光粼粼、倒影清晰）[citation:5] |
| **马卡龙色系** | 《Islanders》清新明亮，粉色樱花岛、绿色草原岛，色彩纯粹愉悦[citation:7] |

### 借鉴建议
为开放世界设计**时间敏感的色彩系统**：清晨偏冷蓝色调、黄昏暖橙色调、夜晚深蓝+局部暖光。水的元素是低模风格中最容易出彩的视觉锚点。

## 五、动画设计

### 关键手法

| 手法 | 说明 |
|------|------|
| **夸张的物理反馈** | 结构不合理时，砖块以 Q 版动画形式散落或塌陷，将负面反馈转化为趣味性体验（《城市规划大师》类游戏）[citation:1] |
| **环境生命力** | 非玩家角色动物（白鸽、海鸥、萤火虫）让城市"活"起来[citation:5] |
| **流畅的过渡动画** | 模块"咔嗒"拼合动画、地基"生长"动画，提供即时愉悦的视觉奖励[citation:1] |
| **即时生成动画** | 建筑"即时生成"，离线时系统自动积累资源，回归时通过加速动画展示成果，形成正向循环[citation:1] |

## 六、交互设计

### 关键手法

| 手法 | 说明 |
|------|------|
| **直觉化操作** | 点击生成地基、长按调色、拖拽旋转视角；复杂操作封装在极简交互逻辑背后[citation:5] |
| **ASMR 级音效** | 模块拼接的"咔嗒"声、水流声、鸟鸣声经过精心设计，形成放松效果[citation:1] |
| **得分驱动的放置玩法** | 《Islanders》：建筑之间位置关系决定分数，实时显示放置每个建筑能获得的分数，在"规划"和"解谜"中找到乐趣[citation:7] |
| **隐藏内容彩蛋** | 建筑组合触发隐藏机关（如三栋房屋围合生成喷泉），增强探索惊喜感[citation:5] |

## 七、代表性游戏案例

### 1. Townscaper
- **类型**：纯建造/无目标玩具
- **核心特色**：无网格自由建造，点击即生成建筑，系统自动根据周围环境生成灯塔/拱门/花园
- **色彩**：明亮饱和，随时间/主题变化（暖橙到梦幻紫）
- **交互**：极简——点一下添加，点一下移除
- **借鉴点**：**自动生成逻辑**极大降低玩家创作门槛

### 2. Islanders
- **类型**：得分驱动的空间解谜
- **核心特色**：建筑放置获得分数，达到阈值解锁新岛屿；无资源管理、无战斗
- **色彩**：马卡龙色系，粉色樱花岛、绿色草原岛[citation:7]
- **交互**：实时显示每个放置位置的得分，"规划=解谜"
- **借鉴点**：将城市建造**数据化、策略化**，满足硬核规划玩家

### 3. Water Town
- **类型**：水系主题自由建造
- **核心特色**：水系为核心，六边形地基系统，建筑可延伸至水下，建筑组合触发隐藏机关（如三栋房屋围合生成喷泉）
- **色彩**：水彩质感，清新通透，水面波光+倒影
- **交互**：自由铺设河道，AR 模式投射现实
- **借鉴点**：**隐藏内容与彩蛋系统**维持长期探索惊喜感

### 4. The Battle of Polytopia（低模之战）
- **类型**：回合制策略+城市建造
- **核心特色**：每个部落拥有独特地形、动物、树木样式；随机生成地图[citation:4]
- **色彩**：各部落风格高度统一
- **交互**：极简科技树与城市管理系统
- **借鉴点**：**程序化生成**作为基础地形，手工打造核心区域

### 5. Mini Metro
- **类型**：交通系统管理（抽象化）
- **核心特色**：将城市简化为几何点和线，专注于交通系统的动态调整
- **色彩**：极简单色系
- **交互**：拖拽连接站点
- **借鉴点**：**数据可视化**和**动态调整**逻辑——在开放世界中加入"交通流"/"资源流"可视化图层

## 八、对 0-1 项目 3D 世界的借鉴建议

### 1. 建立模块化 + 地标体系
提供一套**标准模块化建筑组件**（墙、屋顶、装饰）保证基本盘，同时设计少数高辨识度的**地标建筑**（灯塔、钟楼、城堡）作为玩家的长期目标和视觉锚点。

### 2. "分区 + 地标"引导法
利用地形（山丘、河流）和道路自然划分区域，通过**视觉遮挡**（树林）和**地标露出**引导玩家探索未知区域。避免信息过载。

### 3. 极致化感官反馈
将每一次建造的"点击-放置-生成"过程包装成**声效（ASMR）+ 动画（弹性、生长）+ 震动**的多重奖励。错误反馈（建筑倒塌）应做得有趣而非惩罚性。

### 4. 性能预算即设计
低多边形顶点数少（单个模型 100-1000 顶点），将节省的性能预算用于：
- 更复杂的全局光照
- 动态天气/昼夜系统
- 实时水面反射/折射

### 5. 加入"意外之喜"机制
不要把所有内容平铺直叙。隐藏：
- 建筑组合配方（特定组合触发特殊建筑）
- 特殊地形彩蛋
- 稀有动物吸引条件

让玩家在探索和实验中获得惊喜，这是维持长期活跃度的有效手段。

### 6. 色彩分时段设计
为不同时间段设计专属色彩调性：
- **清晨**：冷蓝色调
- **黄昏**：暖橙色调
- **夜晚**：深蓝底色 + 窗户暖黄光 + 水面倒影

### 7. 水面是最值得投入的视觉元素
低模风格中，水面（反射、倒影、波光）是性价比最高的"氛围感"投资，建议优先实现。
