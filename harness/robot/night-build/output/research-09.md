> 调研平台：DeepSeek（webauth）、Microsoft Learn 官方文档

## 一、核心定位与设计哲学

Windows 11 Widget Board 是一个**侧边滑出式面板**（从屏幕左侧滑入），本质上是一个"个性化 Feed 流 + 快速操作入口"，定位是：

> **"轻量化信息消费 + 无摩擦的操作触发"** — 让用户在不离焦主任务的情况下快速获取信息或启动深层应用。

**官方 Widget 六大原则**（来源：[Microsoft Learn - Widget principles](https://learn.microsoft.com/en-us/windows/apps/design/widgets/)）：

| 原则 | 含义 |
|------|------|
| **Glanceable** | 一瞥即得，用户只需瞄一眼就能获取最大价值，无需点击深入 |
| **Dependable** | 持续显示高频信息，驱动用户重复使用 |
| **Useful** | 呈现最有用、最相关的内容 |
| **Personal** | 个性化内容，建立情感连接；**不允许投放广告** |
| **Focused** | 每个 Widget 聚焦一个主任务，不替代 App/网站 |
| **Fresh** | 内容基于上下文动态刷新，在正确的时间呈现正确的内容 |

## 二、关键设计元素

### 2.1 色彩系统

#### 背景材质：Mica（云母材质）⚠️ 来源：DeepSeek 分析
- **非纯色，非简单高斯模糊**：采集桌面壁纸色调，保持静态模糊
- 效果：让 Widget 面板感觉"从桌面自然浮起"，视觉上连贯，降低认知负荷
- 区别于 Acrylic 的实时动态模糊，Mica 是**静态采样 + 模糊处理**

#### 强调色体系
- 系统级操作（添加按钮、滑块、超链接）使用用户自定义的**系统强调色**
- Widget 内容本身（天气、新闻、股票）使用**高饱和度原生色彩**
- **原则**：系统 UI 绝不与内容争抢视觉焦点

#### 暗色模式适配
- 暗色模式下卡片背景变为深灰（#1F1F1F），字体变为浅灰，Mica 材质同步变暗
- 官方要求：为每种尺寸独立设计亮/暗主题，确保与 OS 无缝融合

#### 无障碍对比度要求
- WCAG 2.0 AA：普通文本 4.5:1，大文本 3:1
- WCAG 2.1 UI 组件 3:1
- WCAG AAA：普通文本 7:1，大文本 4.5:1
- **来源**：[Microsoft Learn - Widget design fundamentals](https://learn.microsoft.com/en-us/windows/apps/design/widgets/widgets-design-fundamentals)

### 2.2 布局系统

#### 整体结构
- Widget Board 从屏幕**左侧滑出**，占据约 35-40% 屏幕宽度（宽屏）
- 形成"侧边仪表盘"形态，右侧留给主屏幕内容

#### 层级结构
1. **顶部栏**：大号时间/日期（强化"当下感"）+ 用户头像（个性化）+ 全屏/展开按钮
2. **搜索栏**：位于顶部下方，**发现新 Widget 的核心流量入口**
3. **主要区域**：**瀑布流式布局**，卡片宽度固定，高度自适应
4. **对齐**：左对齐，右侧留白 — 非对称设计暗示"临时辅助界面"

#### 卡片尺寸系统（官方）
| 尺寸 | 特点 | 触控靶 |
|------|------|--------|
| **小** | 聚焦单一信息，1个操作靶 | 高曝光率 |
| **中** | 2-3 个操作靶，可显示同一信息的更多细节 | 中等曝光 |
| **大** | 更多内容但仍需保持可扫读性；或突出单张图片；**不超过 3-4 个操作靶** | 低曝光率 |

**建议**：所有 3 种尺寸都实现，让用户自由定制

#### 间距规范（官方）
- 卡片外边距：**16px**
- 卡片内 Attribution 区域（顶部）：**48px**（不可放内容）
- 容器间间距（gutter）：**4px**
- 布局数值：**4 的倍数**（保证像素级对齐）

#### Widget 类型
- 每个 Widget 本质上是一个 **Adaptive Card**（JSON 格式定义）
- **来源**：[Microsoft Learn - Widgets terminology](https://learn.microsoft.com/en-us/windows/apps/design/widgets/)

### 2.3 动画系统 ⚠️ 主要来源：DeepSeek 分析

#### 进场/退场动画
- 曲线：**缓动曲线（Ease-Out）** — 模拟物理惯性，非线性
- 时长：**300-400ms**
- 效果：Widget 板从左边缘滑入，主屏幕内容略微向右压缩并变暗（蒙层）

#### 微交互
- **悬停高亮（Reveal Highlight）**：鼠标悬停时卡片边缘出现微弱辉光，指示可交互区域
- **惯性滚动**：滚动列表有物理惯性，滚动条半透明细条，仅滚动时显现

#### 内容更新动画
- **交叉淡变（Crossfade）**：新数据替换旧数据时使用淡入淡出，避免闪烁
- **骨架屏（Skeleton Screen）**：冷启动时显示灰色闪烁占位符，模拟卡片形状

#### 状态记忆
- Widget Board 关闭时如果滚动到某位置，**下次打开会停留在该位置**（而非重置到顶部）

### 2.4 交互系统

#### 呼出方式
| 方式 | 平台 |
|------|------|
| 点击任务栏 Widget 图标 | 键鼠 |
| `Windows + W` 快捷键 | 键鼠 |
| 从屏幕左边缘向右滑动 | 触屏 |

#### 操作靶设计
- 操作按钮（更多选项、刷新等）**默认隐藏**，悬停时才出现（三点菜单图标）
- 平时保持卡片极简，需要时才展开

#### 内置 UI 组件（官方，不可自定义）
- **上下文菜单**：点击右上角三点图标，显示尺寸选择和配置入口
- **归属区域（Attribution Area）**：48px 顶部区域，显示 Widget 名称和图标
- **错误状态**：系统级错误提示 + 刷新按钮；如果有缓存显示"上次更新时间"
- **自定义状态**：用户提供配置模板（Win SDK 1.4+）

#### Widget 状态机（官方）
| 状态 | 说明 |
|------|------|
| **Default** | 正常运行状态，内容完整，不应为空 |
| **Signed-out** | 需要认证的 Widget 可显示非个性化内容或登录入口 |
| **Error** | 系统级错误 UI（名称最多 15 字符截断） |
| **Customization** | 用户可通过配置模板自定义外观/数据 |

#### 工具提示
- 标题文本被截断时使用 tooltip 显示完整内容（正文截断不适用 tooltip）
- **来源**：[Microsoft Learn - Widget states and built-in UI](https://learn.microsoft.com/en-us/windows/apps/design/widgets/widgets-states-and-ui)

#### 头像尺寸规范（官方）
- 支持的圆形头像尺寸：96×96px、48×48px、32×32px、24×24px

## 三、动态信息设计特点

### 3.1 实时更新机制
- 后台**定期拉取数据**，即使界面未打开也会更新缓存
- 打开瞬间看到**最新内容**（非转圈等待）
- 增量更新 + **交叉淡变**动画替换旧数据

### 3.2 个性化 Feed 流
- 集成 Microsoft Start 的 AI 推荐引擎，**非简单时间排序**
- 根据用户阅读习惯、地理位置（天气自动定位）进行智能排序
- 用户可主动过滤来源（"少显示此类内容"/"不再显示此来源"）

### 3.3 自适应卡片
- **高度动态扩展**：如"待办事项"超过 3 条时，卡片高度自适应增加并显示滚动条
- **不挤压其他卡片** — 保持布局独立性
- 不同尺寸下的内容表现：同一 Widget 小尺寸仅显示图标和温度，大尺寸显示未来 5 天预报

## 五、对 0-1 项目 3D 世界的借鉴建议

> 核心原则：**"按需出现，轻量消费"** — 切忌将复杂的 2D 菜单直接堆砌在 3D 空间里

### 5.1 借鉴"悬浮界面"而非"固定 HUD"

**问题**：3D 世界中传统 2D UI（血条、任务列表）破坏沉浸感

**借鉴**：
- **呼出即用，离场即隐**：按 Tab 或特定手势从屏幕边缘呼出"半透明材质"面板
- **Mica 材质移植**：在 3D 世界中，UI 面板背景模糊强度设为 **60-70%**，既能看清文字，又能隐约看到背后 3D 场景动态，保持空间连续性
- **非全屏遮挡**：类似 Widget Board 占据 35-40% 宽度，3D 世界中 UI 面板也应"留边"，不要覆盖整个视野

### 5.2 卡片式组织 3D 资产与任务

**借鉴**：
- **资产卡片化**：3D 模型/道具/NPC 设计为"可悬浮卡片"，正面是渲染缩略图，悬停显示属性
- **动态任务卡片**：卡片边框带**呼吸灯脉冲光**（微弱辉光节奏），吸引对动态变化（如"矿脉即将刷新"、"好友进入世界"）的注意
- **尺寸自适应**：用户可拖动卡片边缘调整大小
  - 小尺寸：仅显示图标（节省视野，适合战斗场景）
  - 大尺寸：显示详细数据（角色状态、装备属性，适合休闲探索）
- **卡片间距**：参考 4px gutter + 16px 外边距，在 3D UI 中保持清爽间距

### 5.3 "被动式信息"与"主动式交互"的平衡

**借鉴**：
- **渐进式披露（Progressive Disclosure）**：
  - 默认只显示简化标题/图标（如仅一个"背包"图标）
  - 准星瞄准（Raycast）或鼠标悬停 **>0.3 秒** 时，才展开详细卡片内容（显示前三项物品）
  - 这避免了 3D 画面被大量按钮遮挡
- **空间音频辅助**：UI 面板呼出时，伴随"全息投影启动"音效，音源定位在面板呼出的屏幕边缘方位，增强临场感

### 5.4 骨架屏应对性能消耗

**问题**：3D 世界 GPU/CPU 消耗极大，UI 加载不能卡帧

**借鉴**：
- **骨架屏优先**：打开面板时数据正在拉取，**不要显示转圈 Spinner**，应立即显示灰色闪烁骨架屏（模拟卡片形状），让用户感知"容器已准备好，内容即将填充"
- **独立渲染层**：3D 世界中的 UI 面板作为**独立 Canvas 渲染层**，即使 3D 场景剧烈运动，UI 面板内滚动仍保持 **120Hz 丝滑响应**，互不干扰

### 5.5 色彩与氛围统一

**借鉴**：
- **环境光遮蔽**：3D 世界中卡片面板边缘应带**自发光（Emissive）** 或 **环境光遮蔽（AO）** 效果，使其看起来像悬浮在 3D 空间中的"实体全息屏"，而非生硬 2D 贴图
- **主题联动**：如果 3D 世界有昼夜循环或特殊天气（下雨、沙尘暴），UI 卡片背景饱和度应相应降低，或增加"水珠划过屏幕"Shader 效果 — **UI 融入环境，而非脱离于环境之上**
- **深色模式优先**：参考 Widget Board 暗色模式，3D 世界的 UI 默认可采用**暗色调 + 高对比度文字**，减少对 3D 场景的色彩干扰

### 5.6 Typography & Touch Targets

**借鉴**：
- 参考 Widget 的字体层级（Segoe UI 变体）：
  - 标题：Extra Large, Bolder
  - 副标题：Large, Bolder
  - 正文：Default / Medium
  - 辅助文字：Small, Lighter
- 触控靶参考 Widget 的 **48px Attribution 区域**，3D 世界中可交互元素最小点击区域不低于 **44×44px**（触屏标准）

### 5.7 状态设计（官方 Widget 原则映射）

| Widget 状态 | 3D 世界对应 | 实现要点 |
|------------|-----------|----------|
| Default | 正常运行态 | 面板内容完整，不应为空白 |
| Signed-out | 未连接/离线态 | 显示简洁的"连接提示"而非报错 |
| Error | 加载失败态 | 友好提示 + 重试按钮；显示缓存数据（如有） |
| Customization | 设置/偏好态 | 用户可自定义面板尺寸、显示内容、透明度 |

## 六、总结

Windows 11 Widget Board 的核心设计哲学：**"按需出现，轻量消费"**

对 0-1 的 3D 世界项目，推荐组合方案：

```
侧边悬浮面板
  + 自适应卡片（呼吸灯边框 + 悬停渐进式披露）
  + 骨架屏加载（异步渲染，不卡 3D 帧率）
  + 主题联动色彩（昼夜/天气影响 UI 饱和度）
  + 独立 Canvas 渲染层（UI 120Hz，3D 场景独立）
```

这套方案能**降低 3D 渲染的视觉负担**，同时在保证功能强大的同时维护 3D 世界的沉浸感与梦幻感。
