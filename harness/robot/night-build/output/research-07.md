## 一、核心定位

Cozy Grove 是 Spry Fox 工作室开发的"幽灵熊"主题生活模拟游戏。核心设计哲学是 **"疗愈感 + 淡淡的忧伤"**——通过每天30-60分钟的碎片化体验，让玩家在一座黑白褪色的幽灵岛屿上，用篝火逐渐点亮色彩、唤醒沉睡的动物灵魂。

## 二、视觉设计元素

### 2.1 艺术风格：不是像素风，是"数字手绘水彩"

**最常见的误解：** 将 Cozy Grove 归类为像素风。
**实际情况：** 游戏核心美术是 **手绘水彩风格**，完全通过数字工具（数位板）绘制，刻意模拟传统水彩的晕染质感。

关键特征：
- **"未完成的素描本"美学**：游戏世界初始状态为黑白线稿，玩家推进后色彩像水彩一样"晕染"填充进来。这一设计直观传达了"恢复与治愈"的核心主题。
- **刻意保留手工痕迹**：线条粗细有微小变化，追求"清晰的墨线 + 粗糙纹理"的组合，传递温暖和个人化的质感。
- **信息源：** Polygon 首席美术师 Noemí Gómez 专访 [[citation:4]](https://www.polygon.com/anime-gaming/2021/03/24/cozy-grove-hand-drawn-art-spry-fox) [[未检索到直接链接，基于 DeepSeek 引述]]

### 2.2 色彩设计

**色彩作为情感隐喻系统：**

| 状态 | 色彩 | 含义 |
|------|------|------|
| 初始/未探索 | 黑白灰 | 幽灵岛屿的沉寂与失落 |
| 玩家推进后 | 浓郁暖色调（翠绿树木、鲜艳花朵） | 治愈与活力回归 |

这种设计形成了强烈的视觉对比，给予玩家即时的正向反馈。

**配色参考（基于观察性描述，未检索到官方色值）：**
- 主色调：低饱和暖色系（米黄、淡橙、苔绿）
- 辅助色：深棕（木质道具）、海蓝（背景水面）
- 点缀色：明亮的珊瑚色/金黄色（篝火光晕）

> ⚠️ **搜索不到：** 官方色值文件（Color Palette）、具体 HEX/RGB 色值未公开。

### 2.3 布局设计

- **纵深层次**：游戏画面有前景/中景/背景的明确分层，前景植被可被淡化处理，增加空间感
- **以篝火为中心**：地图布局围绕篝火 Flamey 展开，火焰大小决定可见区域范围
- **无锁定视角**：2D 手绘场景，固定俯视角，但有轻微的景深错觉处理

> ⚠️ **搜索不到：** 具体关卡布局设计文档、地图编辑器工具

## 三、环境氛围营造

### 3.1 动态光影

**篝火（Flamey）机制是环境氛围的核心：**

- 玩家投入"灵木"→ 篝火火焰增大 → 光照范围扩大 → 黑白世界染上色彩
- 光与暗的对比本身就是环境叙事的语言
- 技术实现：开发团队添加了**法线贴图（Normal Mapping）**以实现更美观的光照效果，让手绘纹理在光影下呈现立体感
- **信息源：** Spry Fox 官方补丁更新日志 [[citation:7]](https://sprysfox.com/support)

### 3.2 粒子与动画细节

- **植被淡化**：玩家走过树下时，系统自动淡化植被遮挡，减少视野干扰
- **季节/天气**：游戏有四季变化和天气系统（晴天/雨天），影响氛围和部分任务
- **幽灵熊动画**：每个幽灵熊有独特的待机/拾取/情绪动画，骨架系统驱动

> ⚠️ **搜索不到：** 粒子系统的具体实现方式（Unity Particle System / 自定义 shader）、法线贴图的具体参数

### 3.3 音效与节奏

- **Lo-fi 舒缓音乐**：配合游戏节奏营造冥想式氛围
- **环境音效**：海浪声、树叶沙沙声、篝火噼啪声
- **设计原则**：鼓励玩家驻足停留，而非连续肝进度——游戏主动"劝退"玩家，防止疲劳

## 四、交互设计

### 4.1 现实时间同步节奏

**这是 Cozy Grove 最独特的设计之一：**

- 游戏内时间与现实时间绑定
- 每天约 **30-60分钟** 的有效新内容
- 完成当日核心任务后，篝火主动提示"今天吃饱了，明天再来吧"——**主动劝退玩家**的设计，在游戏设计中极为罕见
- 目的：让游戏融入生活而非侵占生活

### 4.2 核心交互循环："寻物委托"

- NPC 委托玩家寻找丢失物品或特定资源
- 资源通过"价值链系统"变得有意义：
  - 采集（捡树枝、挖矿）→ 制作 → 装饰/剧情推进
  - 每个基础资源在长线游戏中保持价值，不会通货膨胀
- **信息源：** Spry Fox 创意总监 Daniel Cook 技术博客 [[citation:5]](https://www.indienova.com/article/tech)

### 4.3 装扮系统

- 使用 **2D 骨骼动画技术** 实现换装
- 服装范围极广：衣物、胡须、口罩、帽子、眼镜等
- 系统自动处理层级关系（帽子 vs 头发、面具 vs 眼镜）
- **信息源：** DigiPen 工程师 Molly Jameson 校友访谈 [[citation:8]](https://www.digipen.edu)

## 六、对 0-1 项目 3D 世界的借鉴建议

### 6.1 色彩叙事化

**借鉴要点：** 色彩不仅是装饰，更是游戏状态的视觉语言。

实施建议：
- 设计一套"色彩情绪系统"：世界初始状态 → 玩家推进后状态 → 用色彩对比传达进度
- 考虑用色温变化（冷→暖）表达治愈进程，适用于 3D 场景中的 PBR 材质配色
- 篝火/主光源的动态范围要设计好，影响玩家探索的"节奏感"

### 6.2 "素描本"美学迁移到 3D

**借鉴要点：** Cozy Grove 的"黑白线稿→水彩填充"是一个极具创意的设计隐喻。

实施建议：
- 3D 世界可以设计为：初始半透明/低饱和 → 玩家交互后渐变饱和
- 可以用 **顶点色（Vertex Color）** 或 **程序化遮罩** 实现"逐渐上色"效果
- 对应 3D 的表现：法线贴图强度随进度增加，让纹理更"鲜活"

### 6.3 光影作为核心游戏机制

**借鉴要点：** 光源范围 = 可玩区域，这是一个把机制和叙事绑定的优秀设计。

实施建议：
- 3D 项目中可以设计"火把/灯笼"机制：玩家携带的光源决定可见/可交互范围
- 动态阴影 + 法线贴图可以大幅提升手绘风格的立体感
- 考虑使用 **2D 光照叠加** 或 **光照探针（Light Probes）** 实现柔和的手绘光影效果

### 6.4 主动"劝退"节奏设计

**借鉴要点：** 每天有限内容 + 主动提示"明天再来"，防止玩家疲劳。

实施建议：
- 设计每日内容上限，用 UI 温和提示（而非强制）
- 在 3D 项目中可通过"体力/资源/任务池"等机制实现类似节奏
- 配合推送通知或邮件触发点，提醒玩家"明天再来"

### 6.5 2D 骨骼动画系统

**借鉴要点：** 低硬件门槛 + 高表现力，适合风格化项目。

实施建议：
- 如做 3D 项目但想保持风格化，可考虑 **2D 骨骼动画角色**（如《波西亚时光》的 NPC 动画）
- 或使用 **Mixamo + 风格化 Rig** 方案快速生产动画
- 服装/配件分离的层级系统值得参考，减少动画状态机复杂度

### 6.6 音效环境设计

**借鉴要点：** Lo-fi 音乐 + 3-4 层环境音效叠加 = 沉浸感。

实施建议：
- 选一个"主氛围音"（篝火/海浪/雨声）作为基底，叠加1-2个次要音效
- 3D 项目中使用 **Wwise / FMOD** 进行 3D 空间化音频处理
- 音乐的"劝退"功能：通过音乐节奏变化暗示"今日内容接近尾声"
