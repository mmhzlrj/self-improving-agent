## 一、等距视角实现方式

### 1.1 视角类型：伪等距（Quasi-Isometric）

《动森》采用的并非严格工程制图的等距视角，而是一种**艺术化处理的类等距俯视视角**。

- **非严格轴测投影**：房屋和家具使用轴测投影，消除了近大远小的强烈透视感，保证玩家无论站在房间哪个角落，家具布局都保持稳定可读
- **场景弯曲（曲面化）**：任天堂团队将画面上部的地平线进行**微妙弯曲处理**，模拟鱼眼镜头效果，让远处游玩点仍能被轻松捕捉，增强场景包裹感
- **近景/远景分层**：近景采用俯视视角方便精确移动和摆放物品（如挖洞、种花），远景弯曲则让玩家在行走时自然"撞见"新兴趣点

### 1.2 摄像机控制

- 按右摇杆**上/下**可切换视角仰角（从低角度到近似俯视）
- 提供"经典视角"和"等距视角"两种模式供玩家选择
- 视角切换为无缝过渡，无 loading

### ⚠️ 信息缺失说明

> 任天堂官方未公开房屋编辑器底层 3D 引擎技术文档，以上内容基于第三方技术分析和游戏评测推断。

## 二、可爱风格（Kawaii Style）设计特点

### 2.1 色彩搭配

《动森》色调为**高饱和度、低对比的柔和色板**，大量使用奶油色、马卡龙色和大地色系。

#### 官方调色板（来源：color-hex.com 用户整理）

| 颜色名称 | HEX 值 | RGB | 说明 |
|---------|--------|-----|------|
| 草地绿 | `#9dffb0` | (157,255,176) | 经典草地 |
| 天空蓝 | `#81f1f7` | (129,241,247) | 天空/水面 |
| 米白色 | `#fffffa` | (255,255,250) | 背景/纸张 |

#### 配色策略要点

- **经典组合**：草地鲜绿 + 泥土棕褐 + 天空浅蓝 + 屋顶砖红
- **深色处理**：即使是黑色钢琴/书架也带有柔和灰度，不显沉闷
- **季节主题**：提供 16+ 预设主题（角色主题 + 季节主题），如樱花季、秋季万圣节等

### 2.2 圆角设计

圆角是《动森》UI的灵魂，几乎**没有任何直角边缘**。

| 圆角场景 | 圆角半径 |
|---------|---------|
| 小型按钮/图标 | 8px |
| 标准按钮/卡片 | 12-16px |
| 大型面板/对话框 | 20px+ |
| 家具轮廓 | 高度圆润化处理 |

### 2.3 阴影与扁平化

《动森》采用**"真扁平化"设计**，与主流拟物化/新拟态设计完全相反。

- **无阴影**：不使用阴影来表现层次
- **无渐变**：不使用复杂渐变效果
- **无景深**：不依赖景深效果
- **纯色块区分层级**：界面层级完全依靠纯色块叠加和描边区分
- **噪点/纸纹质感**：UI纹理带有细微噪点或纸纹质感，模拟手工剪纸或水彩纸触感

### 2.4 字体

- 使用**圆体或圆角无衬线字体**
- 标题字体**粗重可爱**
- 正文字体**清晰易读**
- 完美融入圆润整体氛围

## 三、状态显示方式（HUD 元素）

### 3.1 设计哲学：无数值沉浸式

《动森》HUD 设计极度克制，追求**沉浸感优先**。

| 状态类型 | 显示方式 | 交互逻辑 |
|---------|---------|---------|
| **物品耐久** | 无显性数值/进度条 | 工具即将损坏时画面出现细微视觉特效（如渔竿弯曲加剧）+ 音效提示；损坏瞬间才弹出文字提示 |
| **心情数值** | 无显性数值 | 角色状态通过动作反馈表现（快乐时哼歌、无聊时发呆、激动时跳跃） |
| **金币/铃钱** | 屏幕**右上角** | 粗圆体数字 + 金色铃铛图标，设计简洁清晰 |
| **大头菜价格** | 百分比 + 趋势箭头 | 模拟股票交易界面风格，用严肃图表表现轻松内容 |
| **Nook Phone** | 按 **ZL 键** 呼出九宫格 | 模拟智能手机，将复杂功能收纳在统一入口，保持主界面干净 |

### 3.2 Nook Phone 功能列表

（来源：Animal Crossing Wiki / Fandom）

- **DIY 手册**（自定义设计编辑器）
- **地图**（岛屿地图查看）
- **相机**（游戏内拍照）
- **集里游**（里程碑奖励系统）
- **响了**（岛屿音效列表）

### 3.3 设计系统参数（Flutter 复刻库 `animal_crossing_ui` 提取）

第三方 Flutter 组件库对原生动森风格的参数化总结：

- **44+** 自定义组件
- **16+** 预设主题（角色 + 季节）
- **动画时长**：标准过渡 200ms
- **缓动曲线**：支持 `easeOutBack` / `easeOutElastic`（弹窗/按钮点击反馈）
- **Pressed 态**：按下时颜色变暗 10-20%

## 四、整体视觉风格与交互动画特点

### 4.1 动画时长与节奏

| 动画类型 | 时长 | 缓动曲线 |
|---------|------|---------|
| 标准过渡 | 200ms | easeOut（默认）|
| 弹窗弹出 | 200-300ms | easeOutBack（弹性）|
| 按钮点击 | 100-150ms | easeOutElastic（夸张弹性）|
| 角色动作 | 持续循环 | linear（线性）|

### 4.2 弹性与夸张动画

- **菜单弹出**：有轻微的弹性缩放（果冻感）
- **工具切换**：角色有标志性转圈动作（夸张动作既是可爱表现，也是清晰的操作反馈）
- **物品损坏**：视觉特效逐渐加剧（如渔竿弯曲加剧）

### 4.3 多感官反馈

- **视觉**：实时变化的道具形态和特效
- **听觉**：踩在不同地板（草地、木板、地毯）有不同音效
- **触觉**：手柄微弱震动（如手持捕虫网路过花朵时）

### 4.4 "游玩点"视觉引导

任天堂在 CEDEC 2020 分享的核心设计理念：

- **符号化设计**：草地被抽象为三角形纹理，云朵被简化为蓬松块状
- **信息减法**：主动舍弃不必要的细节，减少视觉噪音
- **留白哲学**：画面"不能说得太多"，留给玩家用自己的记忆和想象填补细节

## 六、对 0-1 项目 3D 世界的借鉴建议

### 6.1 视角与相机

| 建议 | 具体做法 |
|-----|---------|
| **采用轴测投影** | 消除强烈透视感，保证 UI 布局稳定性 |
| **远景弯曲处理** | 画面上部地平线微弯，增强场景包裹感 |
| **提供视角切换** | 预留经典视角/等距视角切换选项，满足不同玩家偏好 |

### 6.2 色彩系统

| 建议 | 具体做法 |
|-----|---------|
| **建立色板令牌** | 定义 4-8 种核心色值（草地绿、天空蓝、米白、暖黄、泥土棕等），支持季节/主题切换 |
| **高饱和低对比** | 避免刺眼荧光色，采用柔和马卡龙色系 |
| **深色带灰度** | 暗色元素不要纯黑，带入柔和灰度 |

### 6.3 圆角系统

| 建议 | 具体做法 |
|-----|---------|
| **定义圆角令牌** | 设定 4px、8px、16px、24px 四档，强制所有组件调用 |
| **家具轮廓圆润** | 不仅 UI 组件，游戏内物体（桌子、椅子）也采用圆润边角 |

### 6.4 扁平化层级

| 建议 | 具体做法 |
|-----|---------|
| **去掉阴影** | 不使用阴影，用纯色块 + 描边区分层级 |
| **去掉渐变** | 不使用复杂渐变，用颜色明度区分前后关系 |
| **添加纸纹质感** | UI 背景添加细微噪点/纸纹，模拟手工质感 |

### 6.5 HUD 设计

| 建议 | 具体做法 |
|-----|---------|
| **用体感和音效代替数值** | 物品耐久用视觉特效+音效提示，不显示进度条 |
| **货币显示简洁** | 屏幕右上角大字 + 图标，不需要复杂面板 |
| **功能入口统一** | 复杂功能收纳在"手机/菜单"中，主界面保持干净 |
| **信息减法** | 问自己"去掉细节后玩家还能认出它吗？"——能就去掉 |

### 6.6 动画设计

| 建议 | 具体做法 |
|-----|---------|
| **标准时长 200ms** | 过渡动画不宜过快，保持轻柔感 |
| **使用弹性曲线** | 弹窗/按钮使用 `easeOutBack` 或 `easeOutElastic`，模仿果冻感 |
| **所有可交互元素有 Pressed 态** | 按下时颜色变暗 10-20%，提升操作质感 |
| **夸张动作是反馈** | 工具切换的"转圈"动画既可爱又清晰，要保留 |

### 6.7 设计哲学

| 建议 | 核心理念 |
|-----|---------|
| **符号化 > 拟物化** | 减少视觉噪音，让玩家一眼识别出可交互的"游玩点" |
| **留白 > 塞满** | 画面不能"说得太多"，留给玩家想象空间 |
| **沉浸 > 信息密度** | 能用音效/震动传递的信息，就不要显示在屏幕上 |
