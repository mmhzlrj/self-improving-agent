## 背景

Phase 0-1（语音陪伴）需要：唤醒词检测 + ASR + LLM 对话 + TTS 语音输出。

当前方案（Jetson Nano 软件链路）成本高（¥2000+）、复杂度高。

ESP-AI（`wangzongming/esp-ai`）是开源 AIoT 平台，⭐ 6.9k，将 ASR/LLM/TTS 打包为即插即用组件，ESP32-S3 硬件成本 ¥80-130。

### Step 4: 研究与 Jetson Nano 的通信接口

ESP-AI 支持多种对外接口：

| 接口方式 | 说明 | 适用场景 |
|---------|------|---------|
| UART 串口 | ESP32 通过串口向外发送数据 | Jetson Nano UART 接收 |
| HTTP API | ESP32 调用外部 HTTP 服务 | 与 OpenClaw MCP 集成 |
| WebSocket | 实时双向通信 | 需要低延迟的场景 |

**推荐方案**：UART 串口
```
ESP32-S3 (ESP-AI)
  └── UART TX/RX
        └── Jetson Nano / USB-UART
              └── OpenClaw Node 接收指令
```

定义通信协议：
```
// ESP32 唤醒成功后，通过 UART 发送：
{ "event": "wake", "text": "用户说的内容" }

// Jetson Nano 回复：
{ "event": "response", "text": "LLM 回复文字", "audio_url": "可选TTS URL" }
```

