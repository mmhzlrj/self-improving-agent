### Stage 2: 实施

**方案选择**：轻量文件存储（JSONL），不用额外部署数据库。

**文件结构**：
```
~/.semantic_cache/
├── tasks/
│   ├── tasks_index.jsonl      # 所有 tasks 的索引
│   └── daily/
│       └── 2026-04-02.md      # 每日摘要
```

**实现内容**：

1. **TaskInductor 类**（或独立脚本）：
   - 扫描 sessions 目录获取当天 sessions
   - 提取 session 内容（前 N 轮对话）
   - 调用 LLM 提炼（用 Doubao 或 GLM MCP 工具）
   - 写入 tasks_index.jsonl

2. **每日定时 Cron**：
   - 每天 23:00 CST 执行
   - Cron job 名称：`daily-task-induction`

3. **验证脚本**：
   - 检查 tasks_index.jsonl 格式
   - 检查每日摘要生成

## 验证标准

```bash

# 2. 今天已执行（如果有当天 session）

# 3. 每日摘要文件存在

# 4. Cron 已注册
```
