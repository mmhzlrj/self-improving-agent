## 背景

MemOS Local 是 MemTensor 开发的本地记忆系统（SQLite 存储），功能与当前 Semantic Cache 高度重叠但更完整。需要评估能否替代或增强现有方案。

当前架构：
- Semantic Cache：Ubuntu 节点（192.168.1.18:5050），sentence-transformers + FAISS
- Smart Context Hook：session 前自动注入语义上下文

## 任务目标

1. 安装 MemOS Local 并验证基本功能
2. 对比测试：相同 query 下 MemOS vs Semantic Cache 的召回质量
3. 验证 MemOS 独有功能（Skill Evolution、Task 归纳）
4. 给出决策：替换 / 互补 / 不引入

## 实施步骤

### Step 1: 安装 MemOS Local

```bash
# 方法1：openclaw plugins install
openclaw plugins install github:MemTensor/memos-local

# 方法2：手动 clone
mkdir -p ~/.openclaw/extensions
git clone https://github.com/MemTensor/memos-local ~/.openclaw/extensions/memos-local
```

### Step 2: 配置

在 `~/.openclaw/openclaw.json` 的 `plugins.entries` 添加：

```jsonc
"memos-local": {
  "enabled": true,
  "config": {
    "viewerPort": 18799,
    "embedding": {
      "provider": "openai_compatible",
      "model": "bge-m3",
      "endpoint": "http://127.0.0.1:8080",
      "apiKey": "sk-xxx"
    },
    "summarizer": {
      "provider": "openai_compatible",
      "model": "gpt-4o-mini",
      "endpoint": "https://api.openai.com/v1",
      "apiKey": "sk-xxx"
    }
  }
}
```

**注意**：如果不想额外部署 embedding 服务，可以用 OpenClaw 内置的 embedding（需查看 memos-local 文档确认）。

### Step 3: 重启 Gateway 并验证

```bash
# 重启
openclaw gateway restart

# 验证 Viewer 可访问
curl -s --connect-timeout 5 http://127.0.0.1:18799/
# 期望: 返回 MemOS Viewer 页面
```

### Step 5: 验证独有功能

**Skill Evolution**：
- 在 MemOS 中创建一个测试对话
- 检查是否能归纳为技能
- 检查技能是否能持续升级

**Task 归纳**：
- 多个碎片对话是否被正确归纳为任务

### Step 6: 决策报告

给出以下三种决策之一：

| 决策 | 条件 | 操作 |
|------|------|------|
| **替换** | MemOS 质量 ≥ Semantic Cache + 额外功能更好 | 替换 Ubuntu Semantic Cache |
| **互补** | MemOS 有独特价值，但 Semantic Cache 也不差 | 两个都保留，MemOS 处理跨agent共享 |
| **不引入** | MemOS 质量 < Semantic Cache 或功能重复无增量价值 | 放弃，保持现状 |
