## 一、项目定位

### 核心架构

```
0-1（具身智能机器人项目）
    │
    ├── 身体：具身智能机器人（硬件）
    ├── 大脑：OpenClaw AI 助理（对话层）
    └── 记忆系统：贵庚（Guigeng，底层记忆库）
```

### 三者关系

| 层级 | 定位 | 对外说法 |
|------|------|----------|
| **0-1** | 整体项目 | "我在打造自己的具身智能机器人" |
| **贵庚** | 记忆系统 | Maker圈："是0-1的记忆系统，叫贵庚" |
| **AI助理** | 对话层 | 大众："就是大白那种AI" |

## 二、影响力目标

### 吸引三类人

```
内容输出 → 自然筛选
    │
    ├── 1. 同好：想打造自己具身智能机器人的 Maker
    │           → 组成社群，一起折腾
    │
    ├── 2. 测试需求方：想让你帮忙测试硬件/动作/稀奇古怪的东西
    │           → 你筛选，选择有趣的测试
    │
    └── 3. 投资人：认可这个方向，愿意资金支持
```

### 不追求的事

- ❌ 不追求最大流量
- ❌ 不刻意辐射泛人群
- ❌ 不过度运营粉丝

**原则：** 用内容作为筛选器，不追求说服所有人。

## 四、内容体系

### 三种内容格式

```
原始素材（工作日志/进展）
    ↓
内容生成引擎
    ├── 📝 文字帖子（多平台）
    ├── 🖼️ 配图建议
    └── 🎬 视频脚本（数字人）
```

### 内容节奏

| 素材丰富度 | 处理方式 | 节奏建议 |
|------------|----------|----------|

## 六、技术实现

### 目录结构

```
~/.openclaw/workspace/seed/
├── SEED.md              # 本文档（影响力平台完整说明）
├── SKILL.md             # AI助理使用的技能说明
├── content/
│   ├── raw/             # 原始素材（工作日志）
│   ├── draft/           # 生成的内容草稿
│   └── published/       # 已发布内容
├── assets/              # 素材库（图片/视频片段）
├── queue/               # 发布队列
└── scripts/
    └── generate.sh      # 生成脚本（待实现）
```

### 触发机制

1. **用户触发：** "告诉我今天/这周做了什么" → AI生成内容
2. **自动触发：** cron定期检查新素材

### AI工作流程

```
1. 读取：memory/YYYY-MM-DD.md（当日工作日志）
2. 分析：判断内容丰富度
3. 生成：文字 + 配图 + 视频脚本
4. 输出：到 seed/content/draft/
5. 用户确认 → 发布队列
```
