## 一、贵庚身份系统

### 目标
让贵庚认识用户、记得用户、忘不掉用户、在人群中找到用户

### 技术路径
| 功能 | 技术方案 |
|------|---------|
| 认识我 | 人脸识别（ArcFace）、声纹识别 |
| 记得我 | 语义记忆（贵庚已有）+ 人脸/声纹 embedding |
| 忘不掉我 | 永久存储、多处备份 |
| 找到我 | 行人重识别（ReID）、人脸追踪 |

### 优先级
1. 先做"通过声音认识你"（更简单）
2. 再做人脸识别
3. 最后做ReID追踪

## 二、拍摄机器人系统

### 目标
让机器人学习特定的拍摄技巧，用 iPhone 帮我做多角度的短视频拍摄

### 用户演示场景
1. **希区柯克变焦**：相机后退 + 焦距拉近，背景变大、主体不变（眩晕感）
2. **智能运镜**：自动跟踪、多角度切换

### 技术路径
| 阶段 | 任务 | 技术 |
|------|------|------|
| 1 | iPhone 作为相机节点接入 OpenClaw | OpenClaw iOS bridge |
| 2 | 采集相机运动数据 | 陀螺仪/加速度计 |
| 3 | 用 LeWM 学"运动→画面"关系 | 世界模型 |
| 4 | 复现拍摄手法 | 动作规划 |

### 现状
- LeWM 已在 Ubuntu RTX 2060 上训练完成
- pusht 数据集已下载（7.7GB）
- Vision Encoder 已验证可用

## 三、哨兵守护系统

### 目标
用 ESP32-Cam 做一个守护用户车的哨兵，有人来敲打车或车受损时通知用户

### 原理
```
正常状态：
  当前画面 → LeWM预测 → "下一帧应该和现在差不多"
  → 预测误差小 ✅ 一切正常

异常状态（有人碰车）：
  当前画面 → LeWM预测 → "下一帧应该和现在差不多"
  → 但实际画面完全不一样！
  → 预测误差大 🔴 触发告警
```

### 技术路径
| 优先级 | 任务 |
|--------|------|
| P0 | ESP32-Cam 接入 OpenClaw（RTSP 流） |
| P1 | 采集"正常状态"数据 |
| P2 | LeWM 学习正常状态 |
| P3 | 部署哨兵模式（误差检测 + 告警） |

### 优势
- 比传统移动侦测更准确
- 能识别"不该动的东西被动了"
- 风吹草动不会误报
