## MiniMax Subagent 超时发现（16:20）

**现象**：给 MiniMax subagent 分配 ROBOT-SOP 重复删除+目录补全任务，运行 9m59s 后超时退出（status: timed out）。实际只完成了重复删除部分，目录补全还没开始。

**分析**：
- subagent 的 timeoutSeconds 设了 600（10分钟）
- MiniMax 处理 20 万字符文件需要多轮 read（文件太大一次读不完）
- 重复删除用了约 5 分钟，剩余 5 分钟不够完成目录补全（需要收集 ~90 个 `###` 标题 + 构建 + edit 替换）

**教训**：
- **MiniMax subagent 有硬性 10 分钟超时限制**，超时后任务直接中断，不会继续
- 分配给 subagent 的任务要**评估是否能在此时间内完成**
- 20 万字符文件的 read + 多处 edit = 耗时操作，10 分钟可能不够
- **对策**：大文件任务拆分成多个 subagent，每个只做一部分；或者设更长的 timeout（如果支持）

**后续**：拆成两个 subagent 分别处理（重复删除 ✅ + 目录补全 → 第二个进行中）

### mdview 三级标题渲染修复
- **问题**：Markdown 表格中 `| | | 内容 |` 格式的三级行，Python `markdown.tables` 扩展只认前 2 列，第三个 `|` 后的内容被截断丢失
- **根因**：`markdown` 库解析 `| | | content |` 时认为是 4 列（空、空、content、空），但表格只有 2 列，多出内容被丢弃
- **方案**：检测 `| 章节 | 标题 |` 格式的目录表格，整体转成 HTML `<table>`，根据前导空单元格数量自动判断层级，用 CSS padding 实现缩进
- **修改文件**：`tools/mdview.py`
  - 新增 `preprocess_toc()` 函数：预处理目录表格
  - 新增 `build_toc_html_table()` 函数：构建 HTML 表格
  - 新增 `.toc-table` CSS 样式
- **效果**：章节标题加粗+分隔线，二级标题正常，三级标题缩进+灰色+小字
