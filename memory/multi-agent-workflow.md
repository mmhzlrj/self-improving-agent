# 多 Agents 协同开发工作流

## 核心理念

豆包 + AI Agents 协同解决问题，类似"同事讨论"模式。

## 工作流架构

```
┌─────────────────────────────────────────────────────────────┐
│                     主任务 (Main)                            │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  豆包 (调研)   │   │  Agent A      │   │  Agent B     │
│  联网搜索      │   │  方案设计      │   │  快速试错     │
│  最新信息      │   │  思路探索      │   │  代码验证     │
└───────────────┘   └───────────────┘   └───────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              ▼
                    ┌───────────────────┐
                    │  结果反馈与迭代    │
                    │  (反馈给方案设计)  │
                    └───────────────────┘
                              │
                              ▼
                    ┌───────────────────┐
                    │  死胡同？→ 停止   │
                    │  有进展？→ 继续   │
                    └───────────────────┘
```

## 具体实施

### 1. 调研 Agent（豆包 + Agent）
- 目的：找方案、搜信息、提供思路
- 工具：豆包联网搜索、MiniMax、web_search
- 输出：多个可选方案、关键信息

### 2. 试错 Agent
- 目的：快速验证方案可行性
- 工具：exec、code 执行
- 输出：验证结果、错误信息、可行方案

### 3. 反馈循环
- 试错结果 → 反馈给调研 Agent
- 调研 Agent 根据反馈调整方案
- 继续下一轮试错

### 4. 停止条件
- 多次尝试后无进展
- 出现死胡同（无法绕过的障碍）
- 任务完成

## 使用示例

### 示例：修复一个 Bug

**第1轮**：
```
Agent A（调研）：搜索错误原因，列出3个可能方案
Agent B（试错）：尝试方案1，失败
```

**第2轮**：
```
Agent A：根据失败原因，调整方案
Agent B：尝试方案2，成功
```

**完成**：
```
任务完成，方案已验证
```

## 调度策略

### 什么时候用豆包？
- 联网搜索最新信息
- 简单问答
- 头脑风暴

### 什么时候用 MiniMax？
- 复杂推理
- 长文本生成
- 代码编写

### 什么时候用 Agents？
- 并行调研
- 快速试错
- 多方案验证

## 实现命令

```bash
# 启动调研 Agent
sessions_spawn(runtime="subagent", task="搜索...方案")

# 启动试错 Agent  
sessions_spawn(runtime="subagent", task="尝试执行...")

# 反馈循环
sessions_send(sessionKey="agent:main:subagent:xxx", message="试错结果：...")
```

## 注意事项

1. **不要重复** - 调研和试错分工明确
2. **及时反馈** - 试错结果立即传给调研 Agent
3. **设定上限** - 最多 N 轮后停止，避免死循环
4. **死胡同判断** - 连续 2-3 次失败则停止
