## 09:47 - kimi_kimi_chat 调用调试

### 问题
主 agent 无法直接通过 LLM toolCall 调用 `kimi_kimi_chat`（MCP 工具）。

### 发现
- `alsoAllow` 在 `tools` 级别 + `profile: "full"` 配置正确
- MCP adapter 日志显示 `Registered: kimi_kimi_chat`（11:06:30）
- 但紧接着 `tools.allow allowlist contains unknown entries` 警告（11:06:53）
- 工具注册了但 gateway 不认

### 解决方案
通过 exec 驱动 Node.js MCP server（stdio JSON-RPC）调用：
```bash
echo '{"jsonrpc":"2.0","method":"initialize","params":{},"id":1}
{"jsonrpc":"2.0","method":"tools/call","params":{"name":"kimi_chat","arguments":{"message":"问题","model":"kimi-k2"}},"id":2}' | timeout 120 node ~/.openclaw/extensions/kimi-mcp-server/kimi-mcp-server.mjs
```

### 教训
- MCP 工具调用有时会挂起（Chrome tab 竞争）
- 多个进程同时连接同一个 Chrome tab 会导致冲突
- 失败时应该逐个重试，不要一次性发多个请求
- Kimi thinking 模式超时问题：长问题用 thinking=true 容易超时 120s，短问题正常
