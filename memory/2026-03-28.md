# 2026-03-28 工作日志

## Ubuntu Node 连接 MacBook Gateway

### 目标
让 Ubuntu 台机（192.168.1.18）作为 node 连接到 MacBook Gateway（192.168.1.13）

### 操作步骤

1. Ubuntu 上运行 `openclaw node run --host 192.168.1.13 --port 18789`
2. 报错：SECURITY ERROR - 不允许明文 ws:// 连接
3. 加环境变量：`OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1 openclaw node run --host 192.168.1.13 --port 18789`
4. 报错：ECONNREFUSED - MacBook Gateway 只监听 127.0.0.1，未监听 LAN
5. MacBook 重启 gateway：`openclaw gateway restart`（bind=lan 已设置但需重启生效）
6. Ubuntu 重新连接，报错：pairing required
7. MacBook 上执行 `openclaw devices list` 发现 pending 请求
8. MacBook 上执行 `openclaw devices approve --latest` 批准配对
9. Ubuntu 显示 connected，但 systemd 服务 disconnected
10. 修改 systemd 服务添加 `Environment=OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1`
11. `systemctl --user daemon-reload && systemctl --user restart openclaw-node.service`
12. 成功：jet-Ubuntu paired · connected

### 关键配置

**Ubuntu openclaw.json:**
```json
{
  "gateway": {
    "port": 18789,
    "mode": "remote",
    "bind": "loopback",
    "remote": {
      "url": "http://192.168.1.13:18789",
      "token": "235bf68c2288bfde322b8567509fb0e4d8eb368ec3c70000"
    }
  }
}
```

**systemd 服务文件：** `~/.config/systemd/user/openclaw-node.service`
需添加：`Environment=OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1`

### 教训
- Gateway 配置变更（bind=lan）需要重启才能生效
- systemd 服务不会继承 shell 环境变量，需要在服务文件里显式设置
