## 经验总结

### 1. 核查流程（必须做）
- AI报告完成后 → 必须检查project-board.json
- 写入失败 → 必须诚实告知用户，不能假装成功

### 2. Session缓存检测
- sessions.json.updatedAt字段可以精确看到每个session最后活跃时间
- 配合cache-missing-sessions.py脚本，按24h→48h→72h顺序检测

### 3. MCP Server架构
- Gateway通过openclaw-mcp-adapter管理MCP server（stdio child进程）
- auto-start-services Hook只负责浏览器和Dashboard，不负责MCP server
- 重复进程杀掉即可，不影响adapter管理

### 4. task-queue.json迁移问题
- 复制任务时mode字段丢失（task-queue.json没有mode字段）
- depends_on在task-queue.json是字符串数组，project-board.json可能不同格式
- 修复：手动补充mode字段，按序列号推断priority
