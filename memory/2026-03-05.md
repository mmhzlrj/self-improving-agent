# 2026-03-05 工作日志

## 小龙虾 P2P 聊天网络项目

### 背景
用户想把 OpenClaw 改造成"小龙虾"品牌的去中心化聊天工具，绕过小红书等平台限制，让安装工和业主直接联系。

### 核心需求
- 无服务器 P2P 连接
- 身份证实名（显示自定义昵称）
- 标签匹配（安装工找业主）
- IPv6 直连

### 当前进展

#### 已完成
1. **MVP 开发完成** - P2P 聊天 Skill
   - 文件位置：`~/.openclaw/workspace/skills/lobster-p2p/`
   - 功能：TCP 直连，ping 通就发消息
   - 安装包：`lobster-p2p/pack/lobster-p2p.zip`

2. **技术实现**
   - 端口：5566
   - 协议：TCP + JSON
   - 无需任何中继服务器
   - 支持昵称、标签、联系人管理

3. **用户 IPv6**
   - 240e:3b2:3261:26d0:25f2:745f:7d35:7569

#### 讨论过的问题
- **手机 Node 方案**：手机作为 OpenClaw Node，通过流量 + IPv6 与其他小龙虾 P2P 通讯
- **法律风险分析**：个人/小范围使用风险低；大规模商业化需要牌照
- **P2P 游戏可能性**：基于现有 P2P 模块可开发联机游戏（如斗地主）
- **Blender 游戏引擎**：未来可能整合 Blender 3D 场景 + P2P 同步
- **IPv6 普及**：中国手机端很好（~30%），家庭宽带看运营商；比利时/印度领先
- **6G 影响**：2030 年代会大幅推动 IPv6 物联网设备普及
- **对大厂影响**：微信/钉钉生态护城河深，短期不受影响；运营商流量收入不受影响
