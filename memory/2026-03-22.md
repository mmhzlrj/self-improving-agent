### zhiku MCP 工具使用规范（重要）

**错误做法：** 用 web 搜索代替直接问各平台
- ❌ "这些模型能不能在 iPhone 上跑" → 去搜 web
- ✅ "iPhone AI 目标检测方案" → 用 zhiku-ask.js 问 5 个平台

**正确流程：**
1. zhiku 的 MCP 是用来**直接问各 AI 平台**的工具
2. 不是问"MCP 本身能不能在 iPhone 跑"
3. 是问各平台的**知识和建议**

**zhiku 能问的问题示例：**
- "iPhone 上跑 AI 目标检测有什么方案？"
- "Jetson Nano YOLO 怎么优化？"
- "Genesis 物理引擎最低硬件要求？"

**不能用 zhiku 问的问题：**
- 模型本身的技术细节（应该直接搜文档）
- 纯技术实现问题（应该 web_fetch 官方文档）

### zhiku MCP 超时问题修复经验

**问题：** Kimi/豆包/GLM/千问 page.goto Timeout

**原因分析：**
- goto timeout 太短（15-20s）
- 'load' 等待太久（要等所有资源加载完）
- Qwen textarea 有两个元素，用 `.first()` 选中了只读隐藏的那个

**修复内容：**

| 平台 | 修复前 | 修复后 |
|------|--------|--------|
| Kimi | `goto(..., timeout:20000, waitUntil:'load')` | `goto(..., timeout:45000, waitUntil:'domcontentloaded')` |
| Doubao | `goto(..., timeout:20000, waitUntil:'load')` | `goto(..., timeout:45000, waitUntil:'domcontentloaded')` |
| GLM | `goto(..., timeout:15000)` | `goto(..., timeout:30000)` |
| Qwen | `locator('textarea').first()` | `locator('textarea:not([readonly]):not([aria-hidden="true"])').first()` |

**调试方法：**
1. 先用单个 MCP 工具单独测试，确定哪个平台出问题
2. 确认 MCP server 的 tools/list 能返回正确工具列表
3. 问题定位：先确认 browser 页面能打开，再看 evaluate/拦截器逻辑

**测试命令：**
