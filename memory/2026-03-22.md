### zhiku MCP 超时问题修复经验

**问题：** Kimi/豆包/GLM/千问 page.goto Timeout

**原因分析：**
- goto timeout 太短（15-20s）
- 'load' 等待太久（要等所有资源加载完）
- Qwen textarea 有两个元素，用 `.first()` 选中了只读隐藏的那个

**修复内容：**

| 平台 | 修复前 | 修复后 |
|------|--------|--------|
| Kimi | `goto(..., timeout:20000, waitUntil:'load')` | `goto(..., timeout:45000, waitUntil:'domcontentloaded')` |
| Doubao | `goto(..., timeout:20000, waitUntil:'load')` | `goto(..., timeout:45000, waitUntil:'domcontentloaded')` |
| GLM | `goto(..., timeout:15000)` | `goto(..., timeout:30000)` |
| Qwen | `locator('textarea').first()` | `locator('textarea:not([readonly]):not([aria-hidden="true"])').first()` |

**调试方法：**
1. 先用单个 MCP 工具单独测试，确定哪个平台出问题
2. 确认 MCP server 的 tools/list 能返回正确工具列表
3. 问题定位：先确认 browser 页面能打开，再看 evaluate/拦截器逻辑

### 教训：Gateway 重启会杀 Chrome
**问题**：Gateway restart 会把 Chrome 一起杀掉，导致所有 AI 页面登录状态丢失
**原因**：Gateway LaunchAgent fork 了 Chrome 进程，重启时一并终止

**正确流程**：
1. 先确认 Chrome 调试端口状态
2. 如果 Chrome 已死 → 先重启 Chrome（用正确 profile）→ 再重启 Gateway
3. 或者接受限制：Gateway 重启后必须重新启动 Chrome

### Chrome Debug Profile 记录
| 项目 | 值 |
|------|-----|
| Profile 名 | Chrome-Debug-Profile |
| 路径 | `~/Library/Application Support/Google/Chrome/Chrome-Debug-Profile` |
| 调试端口 | 9223 |
| 用途 | 5个AI页面（豆包/Kimi/GLM/千问/DeepSeek）已登录 |

**启动命令**：
```bash
nohup /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9223 \
  --user-data-dir="$HOME/Library/Application Support/Google/Chrome/Chrome-Debug-Profile" \
  2>/dev/null &
```

**不要做的事**：
- ❌ `--user-data-dir=/tmp/chrome-debug`（空白profile，丢失登录）
- ❌ `osascript -e 'quit app "Google Chrome"'`（关所有窗口）
- ❌ Gateway 重启前不确认 Chrome 状态

### Playwright + Chrome CDP 经验（重要教训）
1. **用 Playwright，不要用 curl**：不能用 curl 操作 Chrome DevTools Protocol 创建标签页
2. **连接已有 Chrome**：用 `chromium.connectOverCDP(wsUrl)`，不要 launch 新浏览器
3. **获取 WS URL**：`curl -s http://127.0.0.1:9223/json/version`
4. **zhiku 用 18800 端口**：旧脚本连接 `http://127.0.0.1:18800`（已废弃，仅参考）

### 重大失误记录（2026-03-22）
1. **偷懒用新 profile**：`--user-data-dir=/tmp/chrome-debug` 创建空白 profile，把用户登录状态全部丢失
2. **用 osascript 关 Chrome**：一刀切关所有窗口，用户正常浏览的页面全丢了
3. **没用 Playwright**：放着现成的 Playwright skill 不用，自己用 curl 乱搞
4. **没有先记录**：用户说"先停下来"的时候我还在执行操作

### Chrome 调试 Profile 管理 SOP
1. Gateway 重启前：先 `curl -s http://127.0.0.1:9223/json/version` 确认端口活着
2. 如果端口断了：杀 Chrome → 重启 Gateway → 启动 Chrome（用 Chrome-Debug-Profile）
3. 杀 Chrome：用 `pkill -f "Chrome.*Chrome-Debug-Profile"` 不要用 osascript
