## 11:41 mdview.py CSS 修复

### 🐛 错误
- **现象**：审批模式 HTML 页面 CSS 被当作纯文本显示在页面上
- **根因**：CSS 变量（`BASE_CSS`、`REVIEW_CSS`）直接用 f-string 插入到 `<head>` 中，没有 `<style>` 标签包裹
- **两处遗漏**：
  1. 普通模式 `generate_normal_html()`：`{BASE_CSS}` 无 `<style>` 包裹
  2. 审批模式 `generate_review_html()`：`{BASE_CSS}` 和 `{REVIEW_CSS}` 无 `<style>` 包裹

### ✅ 修复
- 第1处：`{BASE_CSS}` → `<style>{BASE_CSS}</style>`
- 第2处：`{BASE_CSS}\n{REVIEW_CSS}` → `<style>{BASE_CSS}\n{REVIEW_CSS}</style>`
- 注意：`{REVIEW_JS}` 已经自带 `<script>` 标签，不需要额外包裹

### 💡 教训
1. **f-string 模板中的 CSS/JS 必须检查标签包裹**：用变量插入时容易忘记外层 HTML 标签
2. **旧代码有 `<style>` 包裹是因为写死在字符串里**：重构为变量后丢失了包裹
3. **应该 diff 检查**：重构后用 diff 对比新旧输出，确认结构完整
