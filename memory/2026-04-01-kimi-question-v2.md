## 问题

### 核心问题

**我想把 docs.0-1.ai 打造成类似 docs.openclaw.ai 的专业文档站，请帮我调研以下内容：**

---

#### Q1：如何改造当前架构（Flask + Markdown）使其具有以下现代文档站特性？

需要的功能：
1. **Mintlify 风格的侧边栏 + 搜索 + TOC 目录**
2. **页面内目录（Table of Contents）自动生成**
3. **代码高亮（支持多语言）**
4. **移动端适配**
5. **暗黑/亮色主题切换**

约束：
- 不能破坏现有的 Python Flask 后端
- 不能破坏现有的 Markdown 文件结构
- 保持纯前端增强路线（HTML/CSS/JS）

---

#### Q2：如何实现"调研报告 → 审阅流程 → 自动发布"的工作流？

我希望的工作流：
1. 调研完成后，报告自动出现在 `/reports/pending.html`（待审批列表）
2. 我可以在线审阅（类似 mdview 的审批模式：采纳/拒绝/备注）
3. 审阅完成后，一键发布到对应模块文档或报告区
4. 已发布的报告出现在 `/reports/approved.html`

请给出：
- 前端审阅界面的实现方案
- 后端存储（JSON 文件或嵌入 Markdown 元数据）
- Flask 路由设计

---

#### Q3：如何实现模块版本号 + Release Note 系统？

每个模块（贵庚/LeWM/机械臂/视觉/抓手/移动/面部）需要有：
- 当前版本号（如 v0.1.0）
- 版本历史（changelog）
- 与 OpenClaw 版本号的关联

请给出：
- 数据存储格式（建议 YAML 或 JSON 嵌入 Markdown）
- Release Note 模板
- 如何自动追踪变更（特别是 alsoAllow 类配置变更）

---

我希望：
- 自动生成变更记录（变更时间、变更内容、原因）
