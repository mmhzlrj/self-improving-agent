**调研文件：** `memory/nvidia-ecosystem/README.md`

### 最终价值矩阵
| 技术 | 价值 | 理由 |
|------|------|------|
| NIM + TensorRT-LLM | ⭐⭐⭐⭐⭐ | 本地LLM推理，隐私+离线+高性能 |
| DLSS 5 | ⭐⭐⭐⭐⭐ | 阶段二核心：消费级GPU跑真实感数字孪生 |
| Isaac Lab | ⭐⭐⭐⭐ | 数字孪生训练，需RTX 4080+ |
| cuDNN | ⭐⭐⭐⭐ | PyTorch已自动使用 |
| Isaac ROS | ⭐⭐⭐ | Jetson嵌入式感知 |
| Nsight Systems | ⭐⭐⭐ | 性能诊断，遇到瓶颈再用 |
| DeepStream | ⭐⭐ | Nano已非官方支持，多流优势单机用不上 |
| DLSS 3/4 | ⭐⭐ | 仅 Isaac Sim 仿真渲染时有用 |

### DLSS 5 关键发现
- 2026年秋季发布，定位从"提升帧率"→"重塑画质"
- 核心技术：实时神经网络渲染（AI理解场景后生成像素，不只是补帧）
- RTX 4090 (Gen4)：部分功能，多帧生成不支持
- RTX 5080 (Gen5)：完全支持
- **对0-1的意义**：低分辨率渲染 + DLSS 5 AI重建 → 消费级RTX 4090也能还原真实感数字孪生环境训练贵庚

### DeepStream + Jetson Nano 结论
- Nano已不在官方支持（最后支持DS 7.x）
- 单机单摄像头场景下，MediaPipe + OpenCV 更优
- 建议：升级到 Orin NX/Thor 后再引入DeepStream

### 推荐技术栈（按阶段）
**阶段一（当前）：** NIM + vLLM / MediaPipe + OpenCV / CyberBrick MicroPython
**阶段二（RTX 4080+）：** Isaac Sim + DLSS 5 + Isaac Lab / NIM / TensorRT
**未来（Orin NX/Thor）：** DeepStream / Isaac ROS / TensorRT on Orin

## iPhone 16 Pro + Jetson Nano GPIO 调研（新章节）

### iPhone 16 Pro 作为 OpenClaw 节点
- iPhone 通过 OpenClaw Node 协议接入 Gateway
- 暴露能力：camera/audio/LiDAR/IMU/UWB/NPU(LLM推理)

### Jetson Nano + ESP32-Cam 有线 GPIO 网络
- 两种设备都有 40 针 GPIO（关键洞察）
- 三种有线协议：UART（简单可靠）/ I2C（多设备总线）/ GPIO（应急停止）
- 有线 vs 无线选择：应急停止用 GPIO <1ms，视频流用 WiFi
- 推荐架构：Jetson Nano 作为 I2C 主控，通过 UART/GPIO 控制多 ESP32-Cam

### 完整设备网络拓扑
```
Gateway(MacBook) ← WiFi/MQTT → iPhone 16 Pro(感知前端)
                              ↓ 有线 UART/I2C/GPIO
                         Jetson Nano(控制) → ESP32-Cam×2 + Cyber Bricks
```

## YOLO + MediaPipe 在 Jetson Nano 2GB 上的调研

### 关键发现
- 用户持有的是 Jetson Nano **2GB 版本**（量产模块），非 4GB 开发套件
- 2GB 比 4GB AI 算力低 50%，更容易 OOM

### 实测 FPS（TensorRT FP16）
| 模型 | Nano 4GB | Nano 2GB（预估）|
|------|---------|---------------|
| YOLOv8n | 13-16 FPS | 5-10 FPS |
| YOLOv5n | 15-25 FPS | 8-15 FPS |
| YOLOv11n | 10.6 FPS | 3-7 FPS |

### YOLO + MediaPipe 共存方案
- 两者可以同时跑，但 2GB 很勉强（约 10-15 FPS，需开 swap）
- 推荐串联架构：YOLOv5n 检测人 → MediaPipe 姿态估计
- 2GB 保守策略：只跑一个（MediaPipe ~20 FPS 或 YOLOv5n ~15 FPS）

### Nano 2GB 必开优化
```bash
sudo nvpmodel -m 0 && sudo jetson_clocks
sudo fallocate -l 4G /swapfile && sudo swapon /swapfile
# 输入分辨率降至 320x320
```
