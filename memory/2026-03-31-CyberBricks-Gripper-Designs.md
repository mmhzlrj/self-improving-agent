# Cyber Bricks 抓夹方案：章鱼触手 + 注射器负压

> 调研时间：2026-03-31
> 背景：0-1 机器人在日常生活场景下的抓取方案，利用 H2C 3D 打印能力，实现低成本、高兼容性、完全静音的抓夹系统

---

## 一、设计目标

日常生活场景（家庭环境）的机器人抓取面临的核心挑战：

| 挑战 | 说明 |
|------|------|
| 物体多样性 | 杯子/盒子/手机/钥匙/布料，形状/材质差异大 |
| 精度限制 | Cyber Bricks 开环舵机精度 ±2~5mm |
| 噪音要求 | 家居环境不能有气泵噪音 |
| 成本限制 | 零部件采购尽量少 |

**设计目标**：用 H2C 打印 + 少量采购，实现覆盖大多数日常物体的静音抓夹。

## 二、章鱼触手抓夹方案

### 2.1 核心原理

同时使用两种抓取机制，缺一也能稳住：

```
物体接触
    ↓
吸盘紧贴表面 → 负压吸附（主 grip）
    ↓
触手卷曲缠绕 → 物理包裹（辅助 grip）
    ↓
两种力量同时作用 → 抓取稳定
```

**为什么有效**：吸盘解决「贴上去」，触手缠绕解决「抱得住」。两种机制叠加，同时失效的概率远低于单一机制。

### 2.2 材料选择：TPU 的关键作用

| 普通材料 | TPU（热塑聚氨酯）|
|---------|----------------|
| 硬，无法弯曲 | **柔软可弯曲** |
| 只能打印固定形状 | **可打印曲面、触手形状** |
| 接触面是平的 | **接触面自动贴合物体轮廓** |
| 单点接触 | **面接触，密封性更好** |

H2C 打印 TPU 完全可行（需要低速打印 15~20mm/s，不开风扇）。

### 2.3 具体设计

#### 方案A：单触手 + 多吸盘（最简单）

```
      [吸盘1] [吸盘2] [吸盘3]
         \      |      /
          \     |     /
           \    |    /
            ╰───╯
              |
          绳索牵引
              |
           [舵机]

舵机拉 → 触手卷曲 → 吸盘贴紧 + 物理包裹
```

- 3~4 个小吸盘（直径 10~15mm）
- TPU 打印触手主体（柔性）
- 舵机拉绳索控制卷曲程度
- Cyber Bricks 现有舵机就能控制

#### 方案B：多触手章鱼型（更稳）

```
        [吸盘]     [吸盘]
          \         /
           \       /
      ═══════════════════
      [触手1]    [触手2]
            \   /
             \ /
           [底座]
              |
          [舵机×2]
```

- 2~4 条触手
- 每条触手背面有吸盘
- 舵机分别控制每条触手
- 适合抓圆筒形/球形物体

### 2.4 抓取能力对比

| 物体 | 吸盘单独 | 触手缠绕单独 | 吸盘+触手组合 |
|------|---------|------------|--------------|
| 硬币（平滑）| ✅ | ❌ | ✅ |
| 光滑杯子 | ✅ | ⚠️ 勉强 | ✅ |
| 手机 | ✅ | ⚠️ 勉强 | ✅ |
| 钥匙 | ⚠️ 小吸盘或许 | ✅ | ✅ |
| 布料/衣服 | ❌ | ✅ | ✅ |
| 圆球 | ⚠️ | ✅ | ✅ |
| 盒子 | ✅ | ✅ | ✅ |
| 不规则形状 | ⚠️ | ✅ | ✅ |

### 2.5 TPU 打印参数

| 参数 | 建议值 |
|------|--------|
| 喷嘴温度 | 220~240°C |
| 热床温度 | 50~60°C |
| 打印速度 | **15~20mm/s（必须慢）** |
| 冷却风扇 | **0%（前几层关掉）** |
| 层高 | 0.2mm |
| 填充 | 20~30% |

### 2.6 H2C 能打印的零件

| 零件 | 材料 | 说明 |
|------|------|------|
| 触手主体（曲面）| TPU | 关键件，低速打印 |
| 吸盘（嵌入式）| TPU | 吸盘形状一次成型 |
| 铰链/关节 | TPU | 本身有柔性 |
| 绳索导向槽 | TPU | 打印时成型 |
| 底座（硬质）| PLA | 硬PLA + 软TPU 组合打印 |

**进阶玩法**：PLA 打印硬底座 + TPU 打印触手，分两次打印，螺丝组装。

## 三、注射器负压系统方案

### 3.1 原理

```
电机正转 → 推动注射器活塞 → 排出空气 → 吸盘脱开
电机反转 → 拉回注射器活塞 → 腔内负压 → 吸盘吸附
```

**完全静音**，用步进电机代替气泵。

### 3.2 零件清单

| 零件 | 来源 | 成本 |
|------|------|------|
| 注射器（10~20ml）| 药店/网购 | ~¥0.5 |
| 电机（NEMA 17 42步进）| 现有 Cyber Bricks | ¥0 |
| 电磁阀（常开/常闭）| 可省掉 | ¥0（用电机代替）|
| 硅胶吸盘 | 网购 | ~¥2 |
| H2C 打印件 | 注射器固定座 + 接头 | ¥0 |

**总成本：~¥4**

### 3.3 密封问题

| 方案 | 密封效果 | 能否产生负压 |
|------|---------|------------|
| 直接打印活塞（光滑树脂）| ⚠️ 一般 | ❌ 负压会漏 |
| 在活塞上缠 O 型圈 | ✅ 好 | ✅ 可以 |
| 用橡胶塞（塞进打印件）| ✅ 很好 | ✅ 可以 |

**推荐**：3D 打印件前端装 **O 型圈**（几毛钱一个），解决密封问题。

## 四、完整系统架构

```
┌─────────────────────────────────────────────────────┐
│                  Jetson Nano 2GB                     │
│         （视觉处理：物体识别 + 位置计算）              │
└─────────────────────┬───────────────────────────────┘
                      │ USB / RTSP
                      ▼
┌─────────────────────────────────────────────────────┐
│               ESP32-Cam（OV2640）                   │
│         固定俯视 → OpenCV → 物体定位（±5mm）         │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼ MQTT / UART
┌─────────────────────────────────────────────────────┐
│                 ESP32-C3（Cyber Bricks）             │
│  • 步进电机 A：控制触手卷曲（位置闭环）               │
│  • 步进电机 B：控制注射器抽气/放气（负压控制）        │
│  • AS5600 读数：实时反馈触手位置                     │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│                  章鱼触手抓夹                        │
│  TPU 3D 打印触手（4条）× 硅胶吸盘                    │
│  + 步进电机驱动的注射器负压系统                      │
└─────────────────────────────────────────────────────┘
```

**协同流程**：
1. ESP32-Cam 识别物体位置 → 发给 Jetson Nano
2. Jetson Nano 计算抓取角度 + 触手卷曲程度
3. Cyber Bricks 移动到目标上方
4. 触手接触 → 注射器抽气（负压吸附）→ 舵机收紧触手
5. 移动到目标位置 → 放气 + 松开触手 → 完成
