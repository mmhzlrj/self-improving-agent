已经完成思考
你的代码主要有 3 个关键问题，导致无法获取到元素。此外，还有一些针对“复制按钮”特性的优化建议。
核心问题分析
缺少 await (最致命的问题)
Playwright 的 Python 库是异步的。
locator.all() 是异步方法，需要 await。
locator.is_visible() 也是异步方法，需要 await。
你当前的写法 elements = ...all() 和 if e.is_visible() 会导致程序报错或逻辑错误（协程对象永远为 True）。
get_by_text 的局限性
很多“复制”按钮是 图标 (SVG/Icon)，没有文字“复制”。
文字可能藏在 title 属性里，或者通过 CSS ::before 伪元素生成，get_by_text 无法识别。
按钮可能只在 鼠标悬停 (Hover) 时才显示。
滚动与可见性
scrollIntoViewIfNeeded 是非标准方法（主要是 WebKit），建议用标准 scrollIntoView。
其实 Playwright 的 click() 操作通常会自动滚动到元素位置，手动滚动有时反而多余。
如果内容在 iframe 中，page 对象是找不到的。
