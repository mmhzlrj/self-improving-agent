## 新方案：智库 subagent 实现

### 痛点
- 当前脚本输出合并到一个内容块
- 无法看到独立步骤进度

### 方案
使用 subagent 并行执行，每个 subagent 独立输出到自己的内容块

### 架构
```
用户: 智库 什么是大模型
    │
    ├── subagent-千问 → 千问独立回复块
    ├── subagent-智谱 → 智谱独立回复块
    ├── subagent-Kimi → Kimi独立回复块
    ├── subagent-豆包 → 豆包独立回复块
    └── subagent-DeepSeek → DeepSeek独立回复块
    │
    → 总结块（我合并所有回复）
```

### 关键点
- 5个 subagent 并行运行
- 复用现有浏览器标签页
- 每个步骤/回复独立显示

### AI回复优化方案（来自智库5个平台）

**等待回复完成标记**：
- 不要只等"正在搜索"
- 要等"已完成"、"参考"、"已阅读"、"思考结束"

**复制按钮选择器**：
| 平台 | 选择器 |
|------|--------|
| 千问 | `button[aria-label*="复制"]` 或 `button:has-text("复制")` |
| 智谱 | `button[title*="复制"]` |
| 豆包 | `button[aria-label*="复制"]` |
| Kimi | `button[aria-label*="复制"]` |
| DeepSeek | `button[aria-label*="Copy"], button[aria-label*="复制"]` |

**点击策略**：
- 用 `.last()` 获取最新按钮
- hover回复容器显示隐藏按钮
- 从剪贴板读取：`navigator.clipboard.readText()`
