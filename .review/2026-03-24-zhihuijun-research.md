#### 关键技术亮点（对 0-1 有直接价值）

**① CAN 总线通信（最重要的设计）**
```
优点：
- 4 根线连接所有关节（电源 × 2 + CAN H/L × 2）
- 每个电机有独立 ID，支持广播同步信号
- 支持力矩/速度/位置/轨迹 4 种模式
- 走线极简，适合多关节串联

对比 0-1 现状：
0-1 使用 Cyber Bricks 方案，PWM 控制走线杂乱。
Dummy 的 CAN 总线方案更优雅，是 0-1 升级方向。
```

**② 三种指令模式（对 0-1 运动控制有直接参考价值）**

| 模式 | 特点 | 适合场景 |
|------|------|---------|
| **SEQ**（顺序指令）| FIFO 队列，依次执行 | 视觉抓取、码垛 |
| **INT**（实时指令）| 新指令覆盖旧指令，立即响应 | 动作同步、遥操作 |
| **TRJ**（轨迹跟踪）| 200Hz 插值，精确轨迹 | 3D 打印、绘画、雕刻 |

0-1 可以借鉴这三种模式来实现不同的任务类型。

**③ 零点校准方法（极巧妙，0-1 可以直接用）**
```
传统方式：限位开关 → 归零 → 精度受限
Dummy 方式：
  1. 上电后，电流环控制低力矩定向运动
  2. 碰到机械限位 → 确认为粗零点
  3. 根据单圈绝对值编码器位置精调
结果：精度等同于编码器精度，不受加工误差影响
```

**④ 运动学算法（正逆解，已开源）**
- 使用标准 DH 参数建模
- 逆解：8 组解 → 选最小转角组合（最平滑）
- 梯形加减速曲线进行速度位置规划

**⑤ "青春版"低成本方案（对 0-1 最实用）**

稚晖君计划推出复刻版：
- 3D 打印结构（替代 CNC）
- 自制摆线针轮减速器（替代谐波减速器，¥600 → ¥100）
- 目标成本：**¥2000 以内**
- 配套保姆级视频教程
- 资料待整理发布（参考 GitHub issues 讨论）

#### 稚晖君自制摆线减速器

**GitHub**：https://github.com/peng-zhihui/CycloidAcuratorNano
**现状**：已设计完成，验证中，3D 打印 + 亚克力切割方案

这对 0-1 极具参考价值——如果 Cyber Bricks 的舵机精度不够，可以参考稚晖君的减速器设计自制关节。

## 三、对 0-1 的具体借鉴建议

### 🔴 高优先级（立即可用）

#### ① CAN 总线电机控制 → 替代 Cyber Bricks PWM
**现状**：0-1 用 Cyber Bricks 舵机，PWM 走线杂乱
**稚晖君方案**：CAN 总线，4 线连接所有关节
**行动**：
- 购买支持 CAN 的舵机（如 LDS-3225 或 MG996R + CAN 模组）
- 参考 Dummy-Robot 的驱动固件重构 Cyber Bricks 控制
- 或者直接用稚晖君的开源舵机驱动 XDrive（参考 unlir/XDrive）

#### ② 摆线针轮减速器 → 低成本自制关节
**现状**：Cyber Bricks 舵机精度一般，无法做精细操作
**稚晖君方案**：CycloidAcuratorNano 自制减速器
**行动**：等稚晖君开源后参考，或直接用其设计思路自制

#### ③ 零点校准方法 → 直接用
**无需购买新硬件，只需更新固件逻辑**
- 参考 Dummy 的无限制开关归零算法
- 在 Cyber Bricks 上实现

#### ④ 三指令模式 → 运动控制层升级
**立即可用**，在 MQTT 命令层之下封装 SEQ/INT/TRJ 三种模式

## 六、总结

**稚晖君对 0-1 最有价值的 3 件事**：

1. **CAN 总线电机控制**：4 线替代 PWM 杂乱走线，0-1 可以直接借鉴
2. **运动学算法（正逆解）**：Dummy 已完整开源，0-1 可以直接移植
3. **摆线针轮减速器**：自制低成本高精度关节，Cyber Bricks 升级路径

**最值得等待的开源**：
- Dummy 青春版（¥2000 复刻方案）——稚晖君预告会出保姆级教程
- CycloidAcuratorNano 验证完成后开源

**0-1 的独特优势**：
稚晖君的方案是单机离线为主，0-1 的 P2P Mesh + OpenClaw 分布式架构是所有竞品都没有的。稚晖君解决的是"机器人本身"的问题，0-1 解决的是"机器人联网协作"的问题。
