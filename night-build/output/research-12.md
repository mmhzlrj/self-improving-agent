# Linear 项目管理 UI 调研报告

> 调研时间：2026-03-28
> 平台来源：Exa Search（综合搜索）、Linear 官方 Changelog & Blog、少数派深度分析

---

## 一、概述

Linear 是一款专为软件团队打造的项目管理工具，被设计圈公认为"Linear 风格"的定义者。其 UI 设计在大面积暗色背景下，使用渐变、模糊、动态流光、极细描边、微噪点、外发光以及庄重的无衬线字体，外加流畅克制的微动效来组织和修饰界面元素。产品呈现出冷酷、专业、极简的调性，非常有质感。

> ⚠️ **说明**：本调研基于公开的官方文档、Changelog 和第三方分析文章。未找到 Linear 完整的 Design Token 规范或截图分析，部分设计细节（如具体的动画时长、精确色值）为合理推断，已在文中标注。

## 二、关键设计元素

### 2.1 色彩系统

#### 深色主题基调

| 元素 | 设计处理 |
|------|----------|
| 背景色 | 纯黑或近黑（`#000000` 或 `#0D0D0D` 级别） |
| 表面色 | 多层级灰色，从背景到前景：背景 → 面板 → 卡片 → 对话框，逐层提亮 |
| 强调色 | 淡紫色渐变（Linear Logo 同款），蓝色用于操作按钮和链接，但**克制使用** |
| 文字色 | 高对比度：主要文字白色/近白，次要文字灰色，带透明度的层级文字 |
| 边框/分隔线 | 极低对比度边框（`rgba(255,255,255,0.06)` 级别），几乎"看不见但存在" |

#### 2024 版重大改进：HSL → LCH 色彩空间

**问题**：HSL 色彩空间按数学定义，非人眼感知。同样 50% 亮度，黄色看起来比蓝色亮，导致换主题时颜色失控。

**解决**：换用 **LCH（Lightness-Chroma-Hue）** 感知均匀色彩空间，亮度值 50% 的红色和黄色在人眼中真的一样亮。

| 指标 | 改版前 | 改版后 |
|------|--------|--------|
| 主题变量数 | 98 个 | **3 个**（基础色、强调色、对比度） |
| 主题生成方式 | 手动调整 | 系统自动生成 |
| 高对比度支持 | 需单独设计 | 通过对比度变量**自动生成** |

#### 2026 版 UI Refresh：暖灰方向

- 旧版暗色模式偏冷（蓝色调）
- 新版目标：更**温暖的灰色**，依然保持清爽，但饱和度降低
- 侧边栏略微调暗（`dimmer`），让主内容区更突出
- 蓝色装饰元素的使用被大幅限制

### 2.2 布局结构

#### "倒 L 形" 全局框架

Linear 的布局哲学是"One thing at a time, one dimension at a time"：

- **左侧**：导航侧边栏（可折叠，支持 favorites 和 folders）
- **顶部**：标签页 + 视图标题 + 筛选器（收窄，不再撑满全宽）
- **中间**：主内容区（列表、看板、时间线、分屏、全屏等视图）
- **右侧**：元数据侧面板（Issue 详情、属性）

```
┌─────────┬────────────────────────────────┬──────────┐
│ 侧边栏  │  标签页 + 筛选器 + 视图标题    │          │
│         ├────────────────────────────────┤  侧面板  │
│         │                                │          │
│ (dim)   │         主内容区                │  (issue  │
│         │        (列表/看板/时间线)        │  detail) │
│         │                                │          │
└─────────┴────────────────────────────────┴──────────┘
```

#### 2024 改版重点

| 区域 | 改进 |
|------|------|
| 侧边栏 | 减少视觉噪音，不再凌乱；对齐问题修复（图标/文字/按钮横竖对齐） |
| 标签页 | 更高密度、更好对比度；改为更紧凑的圆角设计 |
| 收件箱 | 行间距收紧（1.5 倍行高保持舒适），一屏显示更多内容 |
| 状态灯 | 不再固定位置，有标签页时会自动调整 |
| 三端统一 | macOS / Windows / 浏览器视觉完全一致，通过 Design Tokens 实现 |

### 2.3 任务列表设计

#### 列表视图

- **极简行条目**：每行一个 Issue，左侧状态指示器 + 标题 + 右侧元数据（ assignee、标签、优先级）
- **分组视图**：支持按状态、负责人、标签等分组显示
- **紧凑模式**：2024 改版后行高收紧，密度提升
- **高级筛选器**：高级筛选菜单，支持组合条件

#### 看板视图

- **泳道**：横向滚动，每个泳道一个状态
- **卡片**：Issue 卡片含状态图标 + 标题 + 优先级 + assignee 头像
- **拖拽**：流畅的拖拽交互，支持在状态间移动

#### 时间线/路线图

- **项目条形**：显示项目周期和里程碑
- **里程碑**：用**菱形标记**（diamond）表示，带目标/完成日期
- **时间刻度**： chronology bar 支持鼠标悬停显示日期线，可切换周数/日期视图
- **折叠逻辑**：靠近的里程碑会自动折叠（类似地图应用）
- **拖拽交互**：拖动项目条可调整日期；选中里程碑可单独移动

---

### 2.4 进度展示

#### 项目进度

- **燃尽图（Burnup Chart）**：项目看板视图支持
- **状态分布**：列表视图顶部显示各状态 Issue 数量统计
- **周期视图（Cycles）**：团队工作周期内 Issue 分布

#### Issue 状态进度

- **Time in Status**：悬停任意 Issue 的状态指示器，可查看在每个状态停留的累计时长（2026 年 1 月新增）
- **状态优先级**：可给项目设置优先级，确保最重要项目优先

### 2.5 动画与微交互

#### 克制为本

Linear 的动效哲学是"流畅但克制"——不为炫技，只为功能性：

| 动效类型 | 实现方式 |
|----------|----------|
| **Hover 反馈** | 背景微亮、透明度变化，无夸张形变 |
| **选中态** | 背景色变化 + 左边框指示器 |
| **展开/折叠** | 高度平滑过渡（通常 200-300ms） |
| **拖拽** | 卡片跟随光标，悬停目标区域高亮 |
| **列表滚动** | 内容单向滚动，无视差效果 |
| **状态切换** | 颜色/图标淡入淡出（opacity transition） |

#### 标志性动效（2019 官网）

- Hero Image 应用图标下出现**激光开场动效**（类似电影开场）
- 渐变背景、发光特效、微噪点均用 CSS `mask-image` + `radial-gradient` + `linear-gradient` 实现
- 矢量图形全用 SVG，位图用 WebP 格式

#### 2024/2026 改版中的动效原则

- **过渡优先于跳转**：用平滑过渡代替突兀的页面切换
- **功能开关切换**：内部开发工具栏可一键开关新旧 UI，对比差异
- **颜色实验工具**：内置颜色选取器可实时调整 LCH 参数并预览

### 2.6 字体系统

| 用途 | 字体 | 样式 |
|------|------|------|
| 正文/UI | Inter | 标准 weight（400-500） |
| 标题/导航 | Inter Display | 更粗字重、字距更紧 |
| 等宽（代码/ID） | JetBrains Mono 或同类型 | 便于识别数字和字母 |

- 标题和正文统一用 Inter，但通过 `Inter Display` 变体区分层级
- 字号层级：标题 > 正文 > 次要文字，清晰可辨

## 四、对 0-1 项目 3D 世界的借鉴建议

> 以下为基于上述调研的合理设计建议，适用于"3D 世界项目管理/进度可视化"类应用。

### 4.1 色彩系统

**建议**：
- 采用 **OKLCH/OKHSL** 色彩空间（现代浏览器已原生支持），确保亮度和饱和度感知均匀
- 深色主题背景用纯黑或接近纯黑（`#090909` 级别），表面逐层提亮
- 强调色选择：考虑到 3D 世界属性，**可考虑紫色/蓝色渐变**（呼应 Linear），或根据项目/世界主题自定义
- **限制强调色数量**：只在大操作（创建、完成、删除）时使用彩色，导航和信息展示用中性色
- 借鉴 Linear 2026 的做法：设计 3 个基础变量（背景基调、强调色、对比度），系统自动生成完整主题

### 4.2 布局结构

**建议**：
- **主-侧结构**：左侧导航（世界/场景切换），中央主内容区（任务列表/3D 视图），右侧详情面板
- **标签页分组**：项目、任务、路线图、设置等分组，每个标签页内独立视图
- **响应式设计**：移动端可折叠侧边栏，3D 视口自适应剩余空间
- **层级清晰**：标题区 → 工具栏 → 内容区 → 状态栏，每层间距规范统一

### 4.3 任务列表

**建议**：
- **简洁行条目**：状态图标 + 任务名 + 优先级 + 负责人头像 + 截止日期
- **支持多视图切换**：列表/看板/时间线/甘特图，Tab 切换而非下拉选择
- **3D 关联展示**：任务可关联到 3D 世界中的特定对象/区域，hover 时用 AR 标记指示位置
- **批量操作**：支持多选后批量修改状态/负责人，保持 Linear 的高效操作体验
- **高级筛选器**：支持保存筛选预设，方便团队快速筛选

### 4.4 进度可视化

**建议**：
- **里程碑设计**：借鉴 Linear 菱形里程碑标记，在 3D 时间轴上用独特的几何图形表示
- **状态时间追踪**：类似"Time in Status"，显示任务在每个状态停留的时长，可用于分析瓶颈
- **燃尽/燃尽图**：在项目面板中集成，与 3D 世界的进度对应
- **进度条**：采用 Linear 克制风格——细线或小圆点填充，不喧宾夺主
- **整体进度环**：在世界概览页用一个优雅的环形进度图展示整体完成度

### 4.5 动画与微交互

**建议**：
- **克制优先**：3D 世界的动画已经够"重"，UI 层动画要极简——hover 透明度变化、展开高度过渡（200ms ease-out）
- **3D 联动**：当 UI 上选中某个对象时，3D 视口自动 orbit 到该对象位置（带平滑过渡）
- **加载态**：使用骨架屏或低对比度的脉冲动画，不使用旋转 loading 圈
- **拖拽反馈**：任务拖拽时 3D 世界中对应对象也微微上浮或发光提示
- **切页过渡**：视图切换时用 opacity 淡入淡出（150-200ms），不做过场动画

### 4.6 字体与图标

**建议**：
- **字体**：中文正文用 Inter/思源黑体（Noto Sans CJK），数字/代码用 JetBrains Mono
- **标题层级**：主标题用粗字重 + 紧凑字距，子标题用常规字重
- **图标**：使用线性图标风格（Linear 本尊就是），保持 1.5px 描边粗细，统一尺寸
- **3D 标注图标**：当 3D 世界中悬停某个对象时，对应的 UI 任务条也高亮（双向联动）

### 4.7 深色主题适配

**建议**：
- 3D 世界通常本身就偏暗色，UI 需确保在深色背景下有足够对比度
- 借鉴 Linear 从 HSL 升级 LCH 的经验，用感知均匀色彩空间定义所有 UI 颜色
- 支持浅色/深色/高对比度三种模式，Linear 的主题生成器思路可参考
- **3D 场景需单独处理**：3D 视口的光照/曝光参数不应与 UI 主题耦合

### 4.8 关键设计原则（学 Linear）

| 原则 | 在 3D 世界项目中的落地 |
|------|----------------------|
| **一次只做一件事** | 任务详情侧面板和 3D 视口不同时展开大面积弹窗 |
| **信息密度高但不拥挤** | 列表行高紧凑，但留足 padding；3D 世界不堆砌标注 |
| **克制用色** | UI 只在操作反馈时变色，信息展示用灰度 |
| **像素级对齐** | 所有 UI 元素严格对齐网格，3D 标注锚点也对齐场景坐标 |
| **快速迭代 + 大版本重置** | 小功能快速上线，但每 6-12 个月做一次设计债务清理 |

## 五、总结

Linear 的 UI 设计核心是**克制的极简主义 + 感知均匀的色彩系统 + 高密度信息展示**。其设计语言已经形成了独立的风格流派，影响了 Arc、Raycast、Craft 等一批产品。

对于 0-1 的 3D 世界项目管理应用，最值得借鉴的是：
1. **LCH/OKLCH 色彩空间**——让深色主题换肤不失控
2. **"倒 L 形"布局框架**——适合固定导航 + 可变内容 + 详情侧栏的结构
3. **微交互克制原则**——3D 世界本身已经有很多视觉元素，UI 层要极简
4. **里程碑 + 时间线**——对 3D 世界的阶段性目标展示非常适配

> ⚠️ **未验证项**：Linear 未公开完整的 Design Token 文档（如具体 HEX 色值、动画时长曲线），上述具体数值均为基于公开信息的合理推断。如需精确实现，建议自行抓取 Linear 的 CSS 变量或参考其开源的 design tokens。
