# AI 工作流 SOP 测评框架

> 适用于评估 AI 自动化流程的效率、质量、成本、稳定性和可维护性

---

## 一、测评指标体系总览

| 维度 | 权重 | 核心指标数 | 测量方式 |
|------|------|------------|----------|
| 效率 | 25% | 6 | 自动化埋点 + 计时 |
| 成本 | 20% | 5 | API 计费日志 |
| 质量 | 25% | 7 | 自动化评估 + 人工抽检 |
| 稳定性 | 20% | 5 | 监控告警 + 重试率 |
| 可维护性 | 10% | 4 | 代码审查 + 文档完整性 |
| **合计** | **100%** | **27** | — |

## 二、各维度详细指标

### 2.1 效率维度 (Efficiency)

| 序号 | 指标名称 | 定义 | 计算公式 | 目标值 |
|------|----------|------|----------|--------|
| E1 | **总执行时长** | SOP 从启动到完成的总耗时 | `T_end - T_start` | ≤ 基准值 × 1.2 |
| E2 | **阶段平均耗时** | 每个子步骤的平均执行时间 | `Σ阶段耗时 / 阶段数` | ≤ 基准值 |
| E3 | **并行优化率** | 并行执行节省的时间比例 | `(串行耗时 - 并行耗时) / 串行耗时` | ≥ 30% |
| E4 | **首次响应时间** | 用户输入到首次输出的间隔 | `首Token输出时间 - T_start` | ≤ 5s |
| E5 | **吞吐量** | 单位时间内完成的实例数 | `成功实例数 / 总时长` | ≥ 基准值 |
| E6 | **阻塞率** | 等待外部资源的占比 | `等待时长 / 总时长` | ≤ 20% |

**评分规则：**
- 每项指标达标得 100 分，未达标按比例扣分
- 效率维度得分 = Σ(指标得分 × 权重) / Σ权重
- 权重：E1=25%, E2=20%, E3=15%, E4=15%, E5=15%, E6=10%

---

### 2.2 成本维度 (Cost)

| 序号 | 指标名称 | 定义 | 计算公式 | 目标值 |
|------|----------|------|----------|--------|
| C1 | **API 调用成本** | 大模型 API 实际消耗费用 | `Σ(调用次数 × 单价)` | ≤ 预算 |
| C2 | **Token 消耗量** | 输入 + 输出的 Token 总数 | `输入Token + 输出Token` | ≤ 基准值 |
| C3 | **单实例成本** | 完成一个 SOP 实例的总成本 | `总成本 / 成功实例数` | ≤ $0.50 |
| C4 | **重试成本** | 因失败重试产生的额外费用 | `重试次数 × 单次成本` | ≤ 总成本 10% |
| C5 | **资源空闲成本** | 等待期间闲置资源费用 | `空闲时长 × 资源单价` | ≤ 总成本 5% |

**评分规则：**
- C1-C3 按预算/基准比例评分，超支 20% 以上得 0 分
- C4-C5 按占比评分
- 成本维度得分 = Σ(指标得分 × 权重) / Σ权重
- 权重：25%, 25%, 25%, 15%, 10%

---

### 2.3 质量维度 (Quality)

| 序号 | 指标名称 | 定义 | 计算公式 | 目标值 |
|------|----------|------|----------|--------|
| Q1 | **输出准确率** | 输出符合预期的比例 | `符合预期数 / 总实例数` | ≥ 90% |
| Q2 | **语法错误率** | 输出中存在语法错误的比例 | `错误数 / 总实例数` | ≤ 5% |
| Q3 | **一致性得分** | 同类输入输出一致性 | `Cohen's Kappa` | ≥ 0.8 |
| Q4 | **用户满意度** | 人工评审满意度 | `5分制平均分` | ≥ 4.0 |
| Q5 | **违规率** | 输出违反安全/合规要求的比例 | `违规数 / 总实例数` | ≤ 1% |
| Q6 | **完整性** | 输出包含全部必要元素的程度 | `实际元素数 / 期望元素数` | ≥ 95% |
| Q7 | **重复率** | 输出内容重复的比例 | `重复Token数 / 总Token数` | ≤ 3% |

**评分规则：**
- Q1-Q3, Q6 使用自动化评估脚本
- Q4 需人工抽检（每批 ≥ 20 个样本）
- Q5, Q7 使用规则检测
- 质量维度得分 = Σ(指标得分 × 权重) / Σ权重
- 权重：20%, 15%, 15%, 20%, 10%, 10%, 10%

---

### 2.4 稳定性维度 (Stability)

| 序号 | 指标名称 | 定义 | 计算公式 | 目标值 |
|------|----------|------|----------|--------|
| S1 | **成功率** | SOP 完整执行成功的比例 | `成功数 / 总执行数` | ≥ 95% |
| S2 | **故障恢复时间 (MTTR)** | 平均故障恢复时间 | `Σ恢复时长 / 故障次数` | ≤ 5 min |
| S3 | **错误率** | 执行过程中出现错误的比例 | `错误数 / 总执行数` | ≤ 3% |
| S4 | **超时率** | 超过设定阈值的比例 | `超时数 / 总执行数` | ≤ 5% |
| S5 | **波动系数** | 性能的变异程度 | `标准差 / 均值` | ≤ 0.2 |

**评分规则：**
- S1, S3, S4 直接反映 SOP 可靠性
- S2 反映运维响应能力
- S5 反映输出一致性
- 稳定性维度得分 = Σ(指标得分 × 权重) / Σ权重
- 权重：30%, 20%, 20%, 15%, 15%

---

### 2.5 可维护性维度 (Maintainability)

| 序号 | 指标名称 | 定义 | 计算公式 | 目标值 |
|------|----------|------|----------|--------|
| M1 | **代码覆盖率** | 单元测试覆盖的代码比例 | `覆盖行数 / 总行数` | ≥ 80% |
| M2 | **文档完整度** | 必要文档的完整程度 | `实有文档数 / 期望文档数` | ≥ 90% |
| M3 | **变更响应时间** | 修改一个环节的平均时间 | 实际测量 | ≤ 2 小时 |
| M4 | **模块化得分** | SOP 各环节解耦程度 | 代码审查评分 (1-5) | ≥ 4.0 |

**评分规则：**
- M1, M2 可自动化检测
- M3 需实际变更记录
- M4 需人工评审
- 可维护性维度得分 = Σ(指标得分 × 权重) / Σ权重
- 权重：30%, 25%, 25%, 20%

## 三、短剧生成 SOP 测评流程

### 3.1 短剧生成 SOP 概述

```
输入：主题/关键词 → 剧本生成 → 分镜设计 → 画面生成 → 配音合成 → 视频剪辑 → 输出
```

| 阶段 | 主要任务 | 涉及模型/工具 |
|------|----------|---------------|
| P1 剧本生成 | 生成短剧剧本、台词 | GPT-4 / Claude |
| P2 分镜设计 | 生成场景描述、镜头参数 | GPT-4 + Vision |
| P3 画面生成 | 生成角色/场景图片 | Midjourney / Stable Diffusion |
| P4 配音合成 | 生成角色配音 | ElevenLabs / GPT-SoVITS |
| P5 视频剪辑 | 合成最终视频 | FFmpeg / Pika |

### 3.5 评分标准（综合得分）

| 等级 | 综合得分 | 评价 | 行动建议 |
|------|----------|------|----------|
| A+ | ≥ 95 | 卓越，可直接投入生产 | 保持现状 |
| A | 85-94 | 优秀，稍加优化即可 | 针对性改进 |
| B | 70-84 | 良好，需重点优化 | 优先改进低分维度 |
| C | 60-69 | 合格，需大规模改进 | 重新设计 SOP |
| D | < 60 | 不合格，暂不推荐使用 | 暂停并重构 |

**综合得分计算：**
```
综合得分 = 效率×0.25 + 成本×0.20 + 质量×0.25 + 稳定性×0.20 + 可维护性×0.10
```

### 4.3 量化计算示例

**案例：短剧生成 SOP 效率计算**

```python
# 伪代码：效率指标计算
def calc_efficiency_metrics(trace_data):
    # E1: 总执行时长
    total_duration = (trace_data['sop_end'].time - 
                     trace_data['sop_start'].time).total_seconds()
    
    # E2: 阶段平均耗时
    stage_durations = []
    for stage in ['剧本', '分镜', '画面', '配音', '剪辑']:
        start = trace_data[f'{stage}_start'].time
        end = trace_data[f'{stage}_end'].time
        stage_durations.append((end - start).total_seconds())
    avg_stage_time = sum(stage_durations) / len(stage_durations)
    
    # E3: 并行优化率 (画面生成与配音可并行)
    serial_time = sum(stage_durations)
    parallel_time = max(stage_durations)  # 简化计算
    parallel_rate = (serial_time - parallel_time) / serial_time
    
    return {
        'total_duration': total_duration,
        'avg_stage_time': avg_stage_time,
        'parallel_rate': parallel_rate
    }
```
